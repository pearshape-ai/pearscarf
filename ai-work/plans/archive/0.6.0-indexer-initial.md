# Plan 0.6.0: Indexer Agent & Knowledge Graph
**Date:** 2026-03-01
**Status:** Draft

## Context

System of Record is in place (v0.5.0). Gmail expert writes emails to `records` + `emails` tables. This version adds the knowledge graph tables and the Indexer — a background agent that processes new records, extracts entities and relationships via LLM, and builds a persistent graph.

The Indexer is NOT a session-based agent. It doesn't communicate via the message bus. It runs its own polling loop, reads from `records`, calls the LLM directly, and writes to the graph tables.

## Database Schema Changes (`db.py`)

### New tables

```sql
-- Add indexed flag to existing records table
-- (SQLite ALTER TABLE ADD COLUMN with DEFAULT is safe)
ALTER TABLE records ADD COLUMN indexed INTEGER NOT NULL DEFAULT 0;

-- Entity type registry
CREATE TABLE IF NOT EXISTS entity_types (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    extract_fields TEXT,       -- JSON array of field names
    added_at TEXT NOT NULL
);

-- Graph nodes
CREATE TABLE IF NOT EXISTS entities (
    id TEXT PRIMARY KEY,
    type TEXT NOT NULL,
    name TEXT NOT NULL,
    metadata TEXT,             -- JSON
    created_at TEXT NOT NULL
);

-- Graph edges
CREATE TABLE IF NOT EXISTS edges (
    id TEXT PRIMARY KEY,
    from_entity TEXT NOT NULL REFERENCES entities(id),
    to_entity TEXT NOT NULL REFERENCES entities(id),
    relationship TEXT NOT NULL,
    source_record TEXT REFERENCES records(id),
    created_at TEXT NOT NULL
);

-- Facts (living state)
CREATE TABLE IF NOT EXISTS facts (
    id TEXT PRIMARY KEY,
    entity_id TEXT NOT NULL REFERENCES entities(id),
    attribute TEXT NOT NULL,
    value TEXT NOT NULL,
    source_record TEXT REFERENCES records(id),
    updated_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_entities_type ON entities(type);
CREATE INDEX IF NOT EXISTS idx_entities_name ON entities(name);
CREATE INDEX IF NOT EXISTS idx_edges_from ON edges(from_entity);
CREATE INDEX IF NOT EXISTS idx_edges_to ON edges(to_entity);
CREATE INDEX IF NOT EXISTS idx_facts_entity ON facts(entity_id);
CREATE INDEX IF NOT EXISTS idx_records_indexed ON records(indexed);
```

### ALTER TABLE handling

SQLite doesn't support `ADD COLUMN IF NOT EXISTS`. Use a try/except around the ALTER:

```python
try:
    conn.execute("ALTER TABLE records ADD COLUMN indexed INTEGER NOT NULL DEFAULT 0")
except sqlite3.OperationalError:
    pass  # Column already exists
```

## `indexer.py` — Indexer Agent

### Structure

```python
class Indexer:
    def __init__(self) -> None:
        self._client = anthropic.Anthropic(...)
        self._stop = threading.Event()
        self._thread: threading.Thread | None = None

    def _build_extraction_prompt(self, record: dict, content: str) -> str:
        """Build prompt from entity_types registry + record content."""

    def _extract(self, record_id: str, content: str) -> dict:
        """Send to LLM, get structured JSON with entities/relationships/facts."""

    def _resolve_entity(self, extracted: dict) -> str:
        """Find existing entity or create new one. Returns entity ID."""

    def _process_record(self, record: dict) -> None:
        """Full pipeline: read content, extract, resolve, write graph, mark indexed."""

    def _loop(self) -> None:
        """Poll for unindexed records, process each."""

    def start(self) -> None
    def stop(self) -> None
```

### Extraction prompt

Built at runtime from `entity_types` table:

```
Given this email record, extract all entities, relationships, and facts.
Respond in JSON only.

Entity types to extract:
- person: A human being. Look for names, email addresses, roles, titles. Fields: name, email, role
- company: A business or organization. Look for company names, domains, industries. Fields: name, domain

Record (email_001):
From: Michael Chen <michael@acmecorp.com>
Subject: Q3 Invoice
Body: Hi Gev, attached is the Q3 invoice...

Respond with:
{
  "entities": [{"type": "person", "name": "Michael Chen", "metadata": {"email": "michael@acmecorp.com"}}],
  "relationships": [{"from": "Michael Chen", "to": "Acme Corp", "type": "works_at"}],
  "facts": [{"entity": "Michael Chen", "attribute": "email", "value": "michael@acmecorp.com"}]
}
```

### Processing pipeline

For each unindexed record:
1. Load full content from typed table (e.g. `emails` JOIN `records`)
2. Build extraction prompt
3. Call LLM → parse JSON response
4. For each extracted entity: `find_entity()` or `create_entity()`
5. For each relationship: `create_edge()` using resolved entity IDs
6. For each fact: `upsert_fact()` using resolved entity IDs
7. `UPDATE records SET indexed = 1 WHERE id = ?`
8. Log everything

### Polling

- Poll every 5 seconds for `records WHERE indexed = 0`
- Process one record at a time
- Runs as a daemon thread

### Logging

Uses `pearscaff.log.write()` with agent name `"indexer"` and session `"--"`.
