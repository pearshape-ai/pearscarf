# Plan 0.4.3: Non-blocking REPL with Attribution and Activity Indicator
**Date:** 2026-02-28
**Status:** Draft

## Context

The REPL currently blocks on `click.prompt()` while agents are working. The user can't type, can't switch sessions, and has no idea what's happening behind the scenes. This version fixes three problems: non-blocking input, clear message attribution, and a live activity indicator.

## Architecture

```

├── terminal.py            # NEW: raw terminal I/O with print-above and status line
├── status.py              # NEW: in-memory agent activity registry
├── repl.py                # MODIFIED: use TerminalUI, status polling, formatted attribution
├── agents/
│   └── runner.py          # MODIFIED: emit status updates on message processing
```

## `terminal.py` — Raw Terminal I/O

A `TerminalUI` class that manages character-by-character input in raw terminal mode. This lets us print messages above the prompt line while the user is typing.

```python
class TerminalUI:
    def __init__(self) -> None:
        self._input_buffer: str = ""
        self._prompt: str = ""
        self._status_line: str = ""
        self._lock = threading.Lock()

    def print_above(self, text: str) -> None:
        """Print text above the current prompt, then redraw prompt + partial input."""

    def set_status(self, text: str) -> None:
        """Update the status line between messages and prompt."""

    def clear_status(self) -> None:
        """Remove the status line."""

    def read_line(self, prompt: str) -> str:
        """Read a line of input using raw terminal mode.
        Supports: typing, backspace, Ctrl+C, Ctrl+D.
        Other threads can call print_above() while this blocks."""
```

### How it works

1. `read_line()` puts the terminal in raw mode via `termios`/`tty`
2. Reads one character at a time from stdin
3. Builds up `_input_buffer` and echoes characters
4. On Enter: returns the buffer, restores terminal
5. `print_above()` (called from other threads):
   - Acquires `_lock`
   - `\r\033[K` to clear current line (prompt + partial input)
   - If status line is showing, `\033[A\r\033[K` to clear it too
   - Prints the message text
   - Redraws status line (if any)
   - Redraws prompt + `_input_buffer`

### Display layout

```
[ses_006] worker > Here are your 3 unread emails...    ← messages (print_above)

[ses_006] gmail_expert working... (5s)                  ← status line (set_status)
[ses_006] you >  partial input here                     ← prompt (read_line)
```

## `status.py` — Agent Activity Registry

A lightweight thread-safe module that tracks what each agent is doing right now.

```python
_activity: dict[str, tuple[str, str, float]] = {}
# key: "agent:session" → (agent_name, status_text, start_timestamp)
_lock = threading.Lock()

def set_status(agent: str, session: str, text: str) -> None:
    """Mark an agent as active in a session."""

def clear_status(agent: str, session: str) -> None:
    """Mark an agent as idle."""

def get_activity(session: str) -> tuple[str, str, float] | None:
    """Get the most recent active agent for a session.
    Returns (agent, text, start_time) or None."""
```

The runner calls `set_status("worker", "ses_001", "processing...")` when it picks up a message, and `clear_status("worker", "ses_001")` when `agent.run()` finishes.

## Changes to `repl.py`

Replace `click.prompt()` with `TerminalUI.read_line()`. Add a status polling thread.

### Threading model

```
Input thread (daemon):      reads from stdin via TerminalUI.read_line(), pushes to queue
Poll thread (daemon):       polls bus for messages, calls TerminalUI.print_above()
Status thread (daemon):     every 1s, checks status.get_activity(), calls TerminalUI.set_status()
Main thread:                processes input from queue, handles commands, sends messages
```

Wait — actually simpler: keep `read_line()` on the main thread (it blocks but is interruptible by print_above from other threads). This avoids the complexity of a queue.

```
Main thread:                TerminalUI.read_line() — blocks on stdin, interruptible by ANSI printing
Poll thread (daemon):       polls bus, uses TerminalUI.print_above() to display messages
Status thread (daemon):     every 1s, updates TerminalUI.set_status() with elapsed time
```

### Message attribution format

All messages use consistent format:

```
[ses_006] you > read my latest unread email
[ses_006] worker > Looking into your emails...
[ses_006] worker > Here are your 3 unread emails:
  From: John <john@example.com>
  Subject: Meeting tomorrow
  ...
```

Different-session notifications:

```
--- [ses_003] gmail_expert: Urgent email from investor ---
```

### Status line format

```
[ses_006] worker working... (3s)
[ses_006] gmail_expert working... (8s)
```

Updates every 1 second. Disappears when the active session has no agents working.
