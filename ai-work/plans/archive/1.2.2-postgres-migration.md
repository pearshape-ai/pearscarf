# Plan: 1.2.2 — Migrate SQLite to Postgres

Spec: `ai-work/specs/1.2.2-postgres-migration.md`

## Summary

Replace SQLite with Postgres for all application data. Same tables, same columns, Postgres types where appropriate (JSONB, SERIAL). Add a docker-compose.yml for all three containers (Postgres, Neo4j, Qdrant). Write a one-time migration script. All application code switches from `sqlite3` to `psycopg` with connection pooling.

### Step 3: Rewrite `pearscaff/db.py` — Postgres backend

This is the core change. Replace the entire SQLite layer with psycopg.

**Key differences from SQLite:**

| SQLite | Postgres |
|--------|----------|
| `?` placeholders | `%s` placeholders |
| `sqlite3.Row` (dict-like) | `psycopg.rows.dict_row` |
| `conn.executescript()` | `cur.execute()` per statement (or single multi-statement string) |
| `PRAGMA journal_mode=WAL` | Not needed (MVCC built-in) |
| `PRAGMA busy_timeout=5000` | Not needed (connection pool handles contention) |
| `INTEGER PRIMARY KEY AUTOINCREMENT` | `SERIAL PRIMARY KEY` or `GENERATED ALWAYS AS IDENTITY` |
| `TEXT` for JSON | `JSONB` for metadata, extract_fields, data |
| `INSERT OR IGNORE` | `INSERT ... ON CONFLICT DO NOTHING` |
| `INSERT OR REPLACE` | `INSERT ... ON CONFLICT ... DO UPDATE` |
| Thread-local connections | Connection pool (`psycopg_pool.ConnectionPool`) |
| `os.makedirs()` for data dir | Not needed (Postgres manages its own storage) |
| `ALTER TABLE ... ADD COLUMN` migration | Columns in initial schema (no migration needed — fresh Postgres) |

**Connection management:**

Replace thread-local `sqlite3.Connection` with `psycopg_pool.ConnectionPool`:

```python
import psycopg
from psycopg.rows import dict_row
from psycopg_pool import ConnectionPool

_pool: ConnectionPool | None = None

def _get_pool() -> ConnectionPool:
    global _pool
    if _pool is None:
        conninfo = (
            f"host={POSTGRES_HOST} port={POSTGRES_PORT} "
            f"dbname={POSTGRES_DB} user={POSTGRES_USER} password={POSTGRES_PASSWORD}"
        )
        _pool = ConnectionPool(conninfo, min_size=2, max_size=10, kwargs={"row_factory": dict_row})
    return _pool

def _get_conn():
    """Get a connection from the pool. Must be used as context manager."""
    return _get_pool().connection()
```

**Usage pattern change:**

All functions that currently do:
```python
conn = _get_conn()
row = conn.execute("SELECT ... WHERE id = ?", (some_id,)).fetchone()
conn.commit()
```

Must change to:
```python
with _get_conn() as conn:
    row = conn.execute("SELECT ... WHERE id = %s", (some_id,)).fetchone()
    # autocommit on exit, or explicit conn.commit()
```

This is the biggest change — every function in db.py needs the `with _get_conn() as conn:` pattern. The pool returns connections to the pool on `__exit__`.

**Important:** Since `_get_conn()` now returns a context manager, all callers outside db.py that import `_get_conn` directly (store.py, graph.py, indexer.py, cli_memory.py) must also switch to the `with` pattern.

### Step 11: Write migration script

**New file:** `scripts/migrate_sqlite_to_postgres.py`

One-time script that:
1. Connects to existing SQLite database at `data/pearscaff.db`
2. Connects to Postgres
3. Runs `init_db()` to create Postgres schema
4. For each table, reads all rows from SQLite, writes to Postgres
5. Handles type conversions:
   - `indexed` INTEGER 0/1 → BOOLEAN
   - `read` INTEGER 0/1 → BOOLEAN
   - `metadata` TEXT (JSON string) → dict (JSONB)
   - `extract_fields` TEXT (JSON string) → list (JSONB)
   - `data` TEXT (JSON string) → dict (JSONB)
6. Validates row counts match
7. Prints summary

Table migration order (respects foreign keys):
1. `sessions`
2. `messages`
3. `discord_threads`
4. `records`
5. `emails`
6. `entity_types`
7. `entities`
8. `edges`
9. `facts`

```python
#!/usr/bin/env python3
"""One-time migration: SQLite → Postgres."""

import json
import sqlite3
import sys

import psycopg
from psycopg.rows import dict_row

from pearscaff.config import (
    POSTGRES_DB, POSTGRES_HOST, POSTGRES_PASSWORD, POSTGRES_PORT, POSTGRES_USER,
)
from pearscaff.db import init_db

SQLITE_PATH = "data/pearscaff.db"

TABLES = [
    "sessions", "messages", "discord_threads",
    "records", "emails",
    "entity_types", "entities", "edges", "facts",
]

# Columns that need JSON string → dict/list conversion for JSONB
JSON_COLUMNS = {
    "messages": ["data"],
    "entities": ["metadata"],
    "entity_types": ["extract_fields"],
}

# Columns that need INTEGER 0/1 → BOOLEAN conversion
BOOL_COLUMNS = {
    "messages": ["read"],
    "records": ["indexed"],
}


def migrate():
    # 1. Connect to SQLite
    sqlite_conn = sqlite3.connect(SQLITE_PATH)
    sqlite_conn.row_factory = sqlite3.Row

    # 2. Init Postgres schema
    init_db()

    # 3. Connect to Postgres
    conninfo = (
        f"host={POSTGRES_HOST} port={POSTGRES_PORT} "
        f"dbname={POSTGRES_DB} user={POSTGRES_USER} password={POSTGRES_PASSWORD}"
    )
    pg_conn = psycopg.connect(conninfo, row_factory=dict_row)

    for table in TABLES:
        rows = sqlite_conn.execute(f"SELECT * FROM {table}").fetchall()
        if not rows:
            print(f"  {table}: 0 rows (skip)")
            continue

        columns = rows[0].keys()
        placeholders = ", ".join(["%s"] * len(columns))
        col_names = ", ".join(columns)
        insert_sql = f"INSERT INTO {table} ({col_names}) VALUES ({placeholders}) ON CONFLICT DO NOTHING"

        json_cols = JSON_COLUMNS.get(table, [])
        bool_cols = BOOL_COLUMNS.get(table, [])

        with pg_conn.cursor() as cur:
            for row in rows:
                values = []
                for col in columns:
                    val = row[col]
                    if col in json_cols and isinstance(val, str):
                        try:
                            val = json.loads(val)
                        except (json.JSONDecodeError, TypeError):
                            pass
                    if col in bool_cols:
                        val = bool(val)
                    values.append(val)
                cur.execute(insert_sql, values)
        pg_conn.commit()

        # Validate
        pg_count = pg_conn.execute(f"SELECT COUNT(*) as c FROM {table}").fetchone()["c"]
        print(f"  {table}: {len(rows)} SQLite → {pg_count} Postgres")

    pg_conn.close()
    sqlite_conn.close()
    print("\nMigration complete.")


if __name__ == "__main__":
    migrate()
```

## Notes

- **Connection management pattern change**: The biggest impact is `_get_conn()` becoming a context manager. Every call site changes from `conn = _get_conn()` to `with _get_conn() as conn:`. This is mechanical but touches many functions across 4 files (db.py, store.py, graph.py, indexer.py, cli_memory.py).
- **JSONB auto-serialization**: psycopg handles Python dict ↔ JSONB transparently. This simplifies code — remove all `json.dumps()` on write and `json.loads()` on read for JSONB columns. The safety check `isinstance(msg["data"], str)` in bus.py becomes a no-op.
- **BOOLEAN vs INTEGER**: SQLite used 0/1 integers for `read` and `indexed`. Postgres uses proper BOOLEAN. Queries change from `read = 0` / `indexed = 0` to `read = FALSE` / `indexed = FALSE`. Python side: `bool(row["read"])` already works — psycopg returns True/False.
- **TIMESTAMPTZ**: Using `TIMESTAMPTZ` instead of `TEXT` for all timestamp columns. The `_now()` function returns ISO strings which Postgres accepts. Could later switch to returning `datetime` objects for richer queries, but ISO strings work fine for now.
- **Migration is optional**: New installs just run `docker compose up -d` and `pearscaff run` — `init_db()` creates everything. Existing users run the migration script first.
- **No schema redesign**: Same tables, same columns, same constraints. Just Postgres types. The spec explicitly says no schema redesign.
- **discord_bot.py**: Imports `db.get_session_by_thread()`, `db.save_thread_mapping()`, `db.get_thread_by_session()` — these function signatures don't change, only their internal implementation. No changes needed in discord_bot.py.
