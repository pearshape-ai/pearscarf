# Plan: 1.2.0 â€” Remove Mem0, Restore SQLite Pipeline

Spec: `ai-work/specs/1.2.0-remove-mem0-storage.md`

## Summary

Remove Mem0 entirely. Restore the SQLite facts + graph + ChromaDB pipeline as the sole storage backend. Remove the `MemoryBackend` abstraction â€” the indexer and retriever go back to using `graph.py`, `vectorstore.py`, and `store.py` directly. Neo4j and Qdrant Docker containers stay in place for future use.

### Step 2: Rewrite `pearscaff/indexer.py` â€” direct SQLite pipeline

**File:** `pearscaff/indexer.py`

Remove the `MemoryBackend` dependency. The indexer goes back to owning its extraction pipeline directly:

1. `_process_record()` builds content (unchanged)
2. Calls `_extract()` â€” LLM extraction via Anthropic (move from old `SqliteBackend._extract`)
3. Resolves entities via `graph.find_entity()` / `graph.create_entity()`
4. Creates edges via `graph.create_edge()`
5. Upserts facts via `graph.upsert_fact()`
6. Embeds in ChromaDB via `vectorstore.add_record()`
7. Marks as indexed

This is essentially restoring the `SqliteBackend.add()` logic into the indexer, minus the `MemoryBackend` wrapper. Keep `trace_span` on `_process_record` and `_extract`.

Imports change:
- Remove: `from pearscaff.memory import MemoryBackend, get_memory_backend`
- Add: `from pearscaff import graph, store`, `from pearscaff.config import ANTHROPIC_API_KEY, MODEL`

Constructor changes:
- Remove `memory: MemoryBackend | None` parameter
- Add `self._client = anthropic.Anthropic(api_key=ANTHROPIC_API_KEY or None)`
