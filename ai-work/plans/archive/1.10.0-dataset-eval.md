## 1. `pearscarf/scoring.py` — Scoring module

Pure functions, no I/O. Each function takes extracted output + ground truth and returns a score.

### Entity matching

```python
def match_entities(
    extracted: list[dict],
    expected: list[dict],
) -> tuple[int, int, int]:
    """Returns (matched, total_extracted, total_expected).

    Match criteria: exact name match (case-insensitive) OR any alias in
    expected_entity['aliases'] matches extracted entity name.
    Type must also match.
    """
```

Matching algorithm:
1. Build a lookup from expected entities: `{canonical_name.lower(): entity, alias.lower(): entity, ...}`
2. For each extracted entity, check `name.lower()` against the lookup. If match found AND `type` matches, count as matched.
3. Each expected entity can only match once (greedy, first-come).

### Fact matching

```python
def match_facts(
    extracted: list[dict],
    expected: list[dict],
) -> tuple[int, int, int]:
    """Returns (matched, total_extracted, total_expected).

    Match criteria: category must be equal, AND from_entity names match
    (case-insensitive), AND to_entity names match (case-insensitive,
    both None counts as match).
    """
```

Matching algorithm:
1. For each extracted fact, look for an expected fact where:
   - `category` is identical
   - `from_entity` matches (case-insensitive)
   - `to_entity` matches (case-insensitive, `None == None`)
2. Each expected fact can only match once.

### Aggregate metrics

```python
def precision(matched: int, total_extracted: int) -> float:
    """matched / total_extracted. Returns 1.0 if total_extracted == 0."""

def recall(matched: int, total_expected: int) -> float:
    """matched / total_expected. Returns 1.0 if total_expected == 0."""

def f1(prec: float, rec: float) -> float:
    """2 * prec * rec / (prec + rec). Returns 0.0 if both are 0."""

def noise_rejection_rate(results: list[dict]) -> float:
    """noise_records_with_zero_extractions / total_noise_records.
    Returns 1.0 if no noise records exist."""
```

`results` is a list of per-record dicts, each carrying `is_noise: bool` and `extracted_empty: bool`.

### Entity Resolution Accuracy

```python
def entity_resolution_accuracy(
    resolution_pairs: list[dict],
    extracted_entities_by_record: dict[str, list[dict]],
) -> float | None:
    """Checks whether surface form variants resolve to the same canonical
    name in extraction output (merge) or correctly stay separate (split).

    Each pair: {"ref_a": str, "ref_b": str, "expected": "merge"|"split",
                "record_a": str, "record_b": str}

    Returns correct_decisions / total_pairs, or None if no pairs.
    """
```

Algorithm:
1. For each pair, find the canonical entity name that `ref_a` resolved to in `record_a`'s extraction, and same for `ref_b` in `record_b`.
2. If `expected == "merge"`: correct if both resolve to the same canonical name.
3. If `expected == "split"`: correct if they resolve to different canonical names.

### Temporal Accuracy

```python
def temporal_accuracy(
    temporal_assertions: list[dict],
    extracted_facts_by_record: dict[str, list[dict]],
) -> float | None:
    """Checks whether extracted facts carry the correct valid_at.

    Each assertion: {"record_id": str, "fact_category": str,
                     "from_entity": str, "expected_valid_at": str,
                     "expected_invalid_at": str|None}

    Returns correct / total, or None if no assertions.
    """
```

Algorithm:
1. For each assertion, find the matching extracted fact in the record (by category + from_entity).
2. Compare extracted `valid_at` to `expected_valid_at` (date-level comparison, strip time).
3. `invalid_at` check is logged but not scored (extraction doesn't produce `invalid_at`; that's an indexer concern).

### Per-record scoring

```python
def score_record(
    extracted: dict,
    expected: dict,
) -> dict:
    """Score a single record's extraction against ground truth.

    extracted: {"entities": [...], "facts": [...]} — LLM output
    expected: {"expected_entities": [...], "expected_facts": [...],
               "is_noise": bool}

    Returns: {
        "entity_precision": float,
        "entity_recall": float,
        "fact_precision": float,
        "fact_recall": float,
        "is_noise": bool,
        "noise_correctly_empty": bool,  # True if noise record AND zero extractions
        "entity_matched": int,
        "entity_extracted": int,
        "entity_expected": int,
        "fact_matched": int,
        "fact_extracted": int,
        "fact_expected": int,
    }
    """
```

## 3. `pearscarf/eval_runner.py` — Dataset loader and orchestrator

### Dataset structure expected

```
<dataset_path>/
├── data/
│   ├── emails/
│   │   ├── email_001.json
│   │   ├── email_002.json
│   │   └── ...
│   ├── issues/
│   │   ├── issue_001.json
│   │   └── ...
│   └── issue_changes/
│       ├── change_001.json
│       └── ...
├── ground_truth.json
├── dimensions.md          # contains dataset version
└── results/               # created by eval runner on first run
    └── 1.10.0_v0.3.0.json
```

### Record file format (`data/<type>/*.json`)

Each file is a single record object matching what the extraction pipeline expects:

```json
{
  "id": "email_001",
  "type": "email",
  "content": "From: john@acme.com\nTo: me@company.com\nSubject: Q3 Invoice\nDate: 2026-03-01\n\nAttached is the Q3 invoice for $13,000.",
  "human_context": null
}
```

`content` is pre-built (same format `Indexer._build_content()` produces). This avoids needing Postgres or type-specific tables. The eval runner feeds `content` directly to the extraction LLM call.

### Ground truth format (`ground_truth.json`)

```json
{
  "version": "0.3.0",
  "records": {
    "email_001": {
      "expected_entities": [
        {"type": "person", "name": "John Chen", "aliases": ["john@acme.com"]},
        {"type": "company", "name": "Acme Corp", "aliases": ["Acme"]}
      ],
      "expected_facts": [
        {
          "category": "WORKS_AT",
          "from_entity": "John Chen",
          "to_entity": "Acme Corp",
          "fact": "John Chen works at Acme Corp"
        }
      ],
      "is_noise": false
    },
    "email_003": {
      "expected_entities": [],
      "expected_facts": [],
      "is_noise": true
    }
  },
  "resolution_pairs": [
    {"ref_a": "john@acme.com", "ref_b": "John Chen", "record_a": "email_001", "record_b": "email_005", "expected": "merge"}
  ],
  "temporal_assertions": [
    {"record_id": "email_001", "fact_category": "COMMITTED_TO", "from_entity": "Acme Corp", "expected_valid_at": "2026-03-01", "expected_invalid_at": null}
  ]
}
```

`resolution_pairs` and `temporal_assertions` are optional top-level keys. If absent, ERA and Temporal Accuracy are skipped.

### Dataset version resolution

```python
def _read_dataset_version(dataset_path: str) -> str:
```

1. Check `ground_truth.json` for `"version"` key → use it.
2. Else check `dimensions.md` for a `version:` field in frontmatter.
3. Else return `"unknown"`.

### Main orchestrator

```python
def run_eval(dataset_path: str) -> None:
    """Load dataset, run extraction on each record, score, report, write results."""
```

Steps:

1. Load `ground_truth.json`, read dataset version.
2. Load record files from `data/` sorted by filename (chronological).
3. Validate: every record file has a matching ground truth entry. Warn on mismatches.
4. Initialize Anthropic client + load extraction prompt (same as `extract_test.py`).
5. For each record:
   a. Build user message: `f"Record ({record_id}, {record_type}):\n\n{content}"`
   b. Call extraction LLM (same model/params as `extract-test`).
   c. Wrap in `trace_span("eval_extract", ...)` with metadata `record_id`, `dataset_version`, `pearscarf_version`.
   d. Parse JSON response.
   e. Score against ground truth via `scoring.score_record()`.
   f. Collect per-record results.
6. Aggregate: sum matched/extracted/expected across all records, compute precision, recall, F1, NRR.
7. If `resolution_pairs` present: compute ERA via `scoring.entity_resolution_accuracy()`.
8. If `temporal_assertions` present: compute Temporal Accuracy via `scoring.temporal_accuracy()`.
9. Print terminal report via `eval_report.print_report()`.
10. Write results JSON via `eval_report.write_results()`.

### LangSmith integration

No new tracing code. The existing `trace_span` from `pearscarf/tracing.py` is used. When `LANGSMITH_TRACING=true`:
- Each record's extraction call is traced as a separate span.
- Metadata tags: `record_id`, `dataset_version`, `pearscarf_version`.
- The project name defaults to `LANGSMITH_PROJECT` from config (user can set `LANGSMITH_PROJECT=pearscarf-eval-v0.3.0` in env to use a dedicated eval project).

### Scope

These metrics cover **extraction quality** — the accuracy of the LLM extraction prompt in producing correct entities and facts from a single record. They are evaluated offline against a synthetic ground truth dataset.

**Not covered (future work):** end-to-end graph quality metrics including graph-level entity resolution (merge/split correctness across the full Neo4j graph), temporal edge integrity after indexer processing, and retriever answer quality. These require a running system and are tracked separately.
