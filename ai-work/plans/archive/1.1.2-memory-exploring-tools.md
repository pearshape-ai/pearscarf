# Plan: 1.1.2 — Memory Exploring Tools

Spec: `ai-work/specs/1.1.2-memory-exploring-tools.md`

## Summary

Add CLI commands and REPL commands to inspect the memory layer — list memories, search, explore entities, view graph stats, and trace memories back to source records. Read-only. Works with both Mem0 and SQLite backends.

---

## Steps

### Step 1: Extend `MemoryBackend` interface with inspection methods

**File:** `pearscaff/memory.py`

Add abstract methods to the backend interface:

```python
class MemoryBackend(ABC):
    # existing
    def add(self, content, metadata) -> None: ...
    def search(self, query, limit) -> list[dict]: ...

    # new
    def get_all(self, limit: int = 10) -> list[dict]: ...
    def get_entity(self, name: str) -> dict | None: ...
    def graph_stats(self) -> dict: ...
    def get_memories_for_record(self, record_id: str) -> list[dict]: ...
```

**Mem0Backend implementations:**
- `get_all()` → `self._mem.get_all(user_id="default", limit=limit)`
- `get_entity()` → Cypher query via Neo4j driver to find node by name, fetch connected nodes and relationships
- `graph_stats()` → Cypher queries: `MATCH (n) RETURN labels(n), count(n)`, `MATCH ()-[r]->() RETURN type(r), count(r)`, most connected nodes
- `get_memories_for_record()` → `self._mem.search(record_id, ...)` filtered by metadata.record_id, or Cypher if needed

**SqliteBackend implementations:**
- `get_all()` → `graph.search_entities("", limit=limit)` + recent facts
- `get_entity()` → `graph.find_entity()` + `graph.get_entity_facts()` + `graph.traverse_graph()`
- `graph_stats()` → SQL counts on entities, edges, facts tables
- `get_memories_for_record()` → SQL: facts/entities where source_record matches

### Step 2: Add Neo4j direct query helper to Mem0Backend

**File:** `pearscaff/memory.py`

Mem0's Python API doesn't expose graph traversal directly. For entity lookup and graph stats, query Neo4j directly using the `neo4j` driver (already installed as a Mem0 dependency).

```python
class Mem0Backend(MemoryBackend):
    def __init__(self):
        ...
        # Keep a direct Neo4j driver for graph queries
        from neo4j import GraphDatabase
        self._neo4j = GraphDatabase.driver(NEO4J_URL, auth=(NEO4J_USER, NEO4J_PASSWORD))
```

This is only for read queries — all writes still go through Mem0.

### Step 3: Create `pearscaff/cli_memory.py` — CLI command group

**New file.** Click command group registered under `cli`.

```python
@cli.group()
def memory():
    """Inspect the memory layer."""

@memory.command("list")
@click.option("--limit", default=10)
def memory_list(limit):
    """List stored memories."""

@memory.command("search")
@click.argument("query")
@click.option("--limit", default=10)
def memory_search(query, limit):
    """Search memories by query."""

@memory.command("entity")
@click.argument("name")
def memory_entity(name):
    """Look up an entity and its connections."""

@memory.command("graph")
@click.option("--limit", default=20)
def memory_graph(limit):
    """Show graph overview and stats."""

@memory.command("record")
@click.argument("record_id")
def memory_record(record_id):
    """Show memories extracted from a specific record."""
```

Each command:
1. Calls `get_memory_backend()`
2. Calls the relevant method
3. Formats output with `click.echo()` + `click.style()` for readability

### Step 4: Register memory commands in `cli.py`

### Step 12: `ps memory list --follow` (tail -f style)

**File:** `pearscaff/cli_memory.py`

Add `-f` / `--follow` flag and `--interval` (default 2s) to `memory_list`:

1. After initial `get_all()` display, collect seen IDs into a set
2. Loop: sleep(interval) → `get_all(limit)` → diff against seen → print new entries with timestamp
3. Ctrl+C to stop

Helper `_get_memory_id(m)` extracts stable ID from memory dict (uses `id` field, falls back to content hash).

Not available in REPL `/memory list` — the REPL must stay responsive to input.

### Step 13: Fix Qdrant exit traceback

**File:** `pearscaff/memory.py`

`QdrantClient.__del__` fires during GC after Python's import system tears down → `ImportError`. Fix:

1. After Mem0 init, import `QdrantClient` and set `QdrantClient.__del__ = lambda self: None`
2. atexit handler does explicit `close()` while Python is still functional
3. Class-level patch covers all instances (Mem0 creates multiple internally)
