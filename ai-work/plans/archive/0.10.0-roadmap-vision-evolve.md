# Plan 0.10.0: Roadmap and Vision Evolution
**Date:** 2026-03-05
**Status:** Draft

## Context

Core agent system works end-to-end: worker, experts, sessions, triage, indexer, retriever. The storage/context layer needs improvement before adding more agents. This version recalibrates the roadmap with focused iterations (0.11.0–0.15.0) covering memory quality, MCP adoption, and observability before new expert agents.

The vision doc also needs rebalancing — the anti-MCP stance is too strong. MCPs, APIs, and headless browsers are all valid transport mechanisms. The real differentiator is the context layer, not the transport.

No code changes. Documentation only.

## `docs/vision.md` — Rewrite with balanced framing

Replace the current content. New structure:

### Core Architecture
- Separation of concerns: experts own domains, worker reasons, storage layer connects everything.
- How an expert accesses its domain (headless browser, MCP, raw API) is an implementation detail. The system doesn't care. Pick the right tool for the job.

### The Context Layer (differentiator)
- The real value is how heterogeneous data gets connected, stored, and surfaced to models when they need it.
- MCPs, APIs, headless browsers are all valid transport mechanisms. The architecture supports swapping between them without touching anything outside the expert.

### How the System Learns
- Keep the existing four learning mechanisms (HIL triage, graph growth, auto-classification, human context enrichment) — these are still accurate.

### Retrieval: Explicit and Implicit
- Keep existing content — still accurate.

### Multi-Expert Future

- Remove the implication that headless browser is the only future.
