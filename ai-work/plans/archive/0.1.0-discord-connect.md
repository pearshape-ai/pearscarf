# Plan 0.1.0: Discord Connect
**Date:** 2026-02-26
**Status:** Draft

## Context

The agent currently only works via CLI REPL (`pearscaff chat`). This adds a Discord bot so the same agent can respond to messages in a Discord server.

## Architecture

```
pearscaff/
├── agent.py           # Refactor: remove print(), add callback for tool logging
├── cli.py             # Add `discord` command
├── config.py          # Add DISCORD_BOT_TOKEN
├── discord_bot.py     # NEW: Discord bot using discord.py, per-channel Agent instances
```

Key decisions:
- Per-channel Agent instances (separate conversation history per channel)
- Run agent in executor thread since Anthropic SDK is sync and discord.py is async
- Tool call logging via optional callback instead of bare print()
- Single new CLI command: `pearscaff discord` / `ps discord`

## Implementation Steps

1. Add `discord.py` to dependencies
2. Add `DISCORD_BOT_TOKEN` to config.py and .env.example
3. Refactor agent.py — replace `print()` with optional `on_tool_call` callback
4. Create `pearscaff/discord_bot.py` — bot client with on_message handler, per-channel agents
5. Add `discord` command to cli.py
6. uv sync and verify

## Verification

```bash
export DISCORD_BOT_TOKEN=...
pearscaff discord  # bot comes online in server
# Send a message in Discord, bot responds
```
