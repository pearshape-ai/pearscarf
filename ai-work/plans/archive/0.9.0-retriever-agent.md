# Plan 0.9.0: Retriever Agent
**Date:** 2026-03-03
**Status:** Draft

## Context

System of Record, Indexer, ChromaDB, and triage are all in place. Data is stored and enriched but the worker can't query it. This version adds the Retriever — an expert agent the worker delegates to when the human asks a question that requires context from the knowledge graph or vector store.

The Retriever is a proper expert agent (uses `ExpertAgent` base, communicates via the message bus, has a `reply` tool). The worker sends it a query, it searches facts + graph + vectors, and returns a structured context package.

## `experts/retriever.py` — Retriever Expert Agent

An `ExpertAgent` with three tools matching the spec's three query modes:

### System prompt

```
You are the retriever expert agent. You find relevant context from the knowledge graph
and vector store.

When you receive a query:
1. Use search_entities to identify if the query references known entities.
2. If entities found, use facts_lookup to get their attributes.
3. Use graph_traverse to find connected entities and source records.
4. Use vector_search for semantically similar records.
5. Assemble the results into a structured context package and reply.

Always reply with a structured summary: facts found, related records, connected entities,
and your reasoning about what was found.
```

### Tools

**`graph_traverse`** — Walk edges from an entity up to N hops:


**`vector_search`** — Semantic similarity search via ChromaDB:

The retriever also has `search_entities` (reused from graph module) to identify entities by name/email/domain before looking up facts.

### Factory

```python
def create_retriever_for_runner(bus: MessageBus) -> callable:
    """Factory for AgentRunner. Returns agent_factory function."""
    def factory(session_id: str) -> ExpertAgent:
        registry = ToolRegistry()
        registry.register(FactsLookupTool())
        registry.register(GraphTraverseTool())
        registry.register(VectorSearchTool())
        registry.register(SearchEntitiesTool())  # reuse from worker or make standalone
        return ExpertAgent(
            domain="retriever",
            domain_prompt=RETRIEVER_SYSTEM_PROMPT,
            tool_registry=registry,
            bus=bus,
            agent_name="retriever",
        )
    return factory
```

Uses the recursive CTE from the spec:
```sql
WITH RECURSIVE connected AS (
    SELECT to_entity, relationship, source_record, 0 as depth
    FROM edges WHERE from_entity = ?
    UNION ALL
    SELECT e.to_entity, e.relationship, e.source_record, c.depth + 1
    FROM edges e JOIN connected c ON e.from_entity = c.to_entity
    WHERE c.depth < ?
)
SELECT DISTINCT * FROM connected;
```
