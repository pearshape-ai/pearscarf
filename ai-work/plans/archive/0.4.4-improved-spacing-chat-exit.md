# Plan 0.4.4: Fix Crash on Exit and Response Spacing
**Date:** 2026-02-28
**Status:** Draft

## Context

Two bugs in the REPL need fixing:
1. Typing `exit` crashes with a greenlet thread error because `gmail_manager.close()` tries to save Playwright browser state from the wrong thread.
2. Worker response text displays with erratic indentation and line breaks.

## Bug 1: Crash on Exit

### Root Cause

`BrowserManager.close()` calls `self.save_state()` which calls `self._context.storage_state(...)`. Playwright contexts are bound to the greenlet/thread that created them. The browser launches lazily on the runner thread (`runner-gmail_expert`), but `close()` is called from the main thread in `cli.py:46`.

### Fix

Wrap `save_state()` in a try/except in `BrowserManager.close()` to catch the greenlet error gracefully. The browser state was already saved on the runner thread during normal operation, so failing to save on shutdown is safe.

```
pearscaff/experts/gmail.py  # MODIFIED: BrowserManager.close() catches greenlet error
```

### Why not fix the threading?

Moving `close()` to the runner thread would require plumbing a shutdown callback through the runner, adding complexity for no real benefit. The state was already persisted during normal browser operations. Catching the error is the simplest correct fix.

## Bug 2: Response Text Spacing

### Root Cause

Multi-line responses from agents (e.g. email summaries with line breaks) get printed as a single `print_above()` call. The raw terminal output doesn't handle embedded newlines well — each line after the first is not properly flushed or aligned relative to the ANSI escape sequences used to clear and redraw the prompt.

### Fix

In `repl.py`'s `_poll_responses`, split multi-line content and print each line with proper attribution on the first line and indentation on continuation lines. This keeps the output flush left and readable.

```
pearscaff/repl.py  # MODIFIED: split multi-line responses, indent continuation lines
```

  - Build the prefix: `[ses_001] worker > `
  - First line gets the full prefix
  - Continuation lines get indented by the prefix width (spaces only, no color codes)
- This keeps multi-line responses aligned and readable
