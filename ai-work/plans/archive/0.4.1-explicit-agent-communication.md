# Plan 0.4.1: Explicit Agent Communication (Kill the Ping-Pong)
**Date:** 2026-02-28
**Status:** Draft

## Context

Agents are stuck in infinite farewell loops. When agent A sends a message to agent B, the runner unconditionally runs `agent.run()` on the response and sends the output back — creating an endless ping-pong of pleasantries. The root cause is in `runner.py:_process_message`: it always calls `bus.send()` after `agent.run()`, regardless of whether the agent has anything meaningful to say.

The fix: **remove the auto-reply from the runner entirely**. All outbound communication becomes explicit via tools. If an agent has nothing to send, it simply doesn't call a send tool, and no message goes out.

## Core Change: Runner No Longer Auto-Replies

**Before** (`runner.py:_process_message`):
```python
response = agent.run(content)
self._bus.send(session_id, self._agent_name, from_agent, response, ...)  # ALWAYS sends
```

**After**:
```python
agent.run(content)  # Agent uses tools to send messages if needed. Text output is logged only.
```

The runner's job becomes: receive message → run the agent → log output → done.

## New Tools

### `send_message` (Worker)

Replaces the implicit auto-reply. Worker uses this to explicitly send messages to the human or any other agent.

```python
class SendMessageTool(BaseTool):
    name = "send_message"
    description = "Send a message to another agent or to the human user."
    input_schema = {
        "type": "object",
        "properties": {
            "to": {"type": "string", "description": "Recipient: 'human', or an expert name"},
            "content": {"type": "string", "description": "The message content"},
        },
        "required": ["to", "content"],
    }
```

This subsumes `delegate_to_expert` — the worker uses `send_message(to="gmail_expert", ...)` to delegate AND `send_message(to="human", ...)` to reply. One tool for all outbound communication.

### `reply` (Expert)

Experts use this to send their result back to whoever asked. The tool knows the `from_agent` of the incoming message (set by the runner before calling `agent.run()`).

```python
class ReplyTool(BaseTool):
    name = "reply"
    description = "Send your response back to the agent that requested this work."
    input_schema = {
        "type": "object",
        "properties": {
            "content": {"type": "string", "description": "Your response"},
        },
        "required": ["content"],
    }
```

## System Prompt Updates

### Worker
Add to `WORKER_SYSTEM_PROMPT`:
```
IMPORTANT: You MUST use the send_message tool to communicate. Your text responses
are only logged internally — nobody sees them unless you use send_message.

- Use send_message(to="human", ...) to respond to the user.
- Use send_message(to="gmail_expert", ...) to delegate tasks to experts.
- Do NOT send thank-you or farewell messages to experts. When you receive results
  from an expert, process them and send_message to human. That's it.
```

### Expert
Add to expert system prompts:
```
IMPORTANT: You MUST use the reply tool to send your results back.
Your text responses are only logged internally — nobody sees them unless you use reply.

- When you finish your task, use reply(content=...) with your results.
- Do NOT send pleasantries, thank-yous, or farewells. Just deliver results.
- Use reply exactly once per request. After replying, your work is done.
```

## Message Flow After Fix

### Human asks to read emails
```
human →(bus)→ worker: "Read my latest emails"
  worker agent runs, calls send_message(to="gmail_expert", content="List unread emails")
  worker agent also calls send_message(to="human", content="Looking into your emails...")

gmail_expert receives message from worker
  expert agent runs, operates browser tools, calls reply(content="3 unread emails: ...")

worker receives expert response
  worker agent runs, calls send_message(to="human", content="You have 3 unread emails: ...")
  worker does NOT send anything back to gmail_expert. Done.
```

### Expert-initiated alert
```
(Expert would create session + send_message to worker — future capability, unchanged)
```

## Implementation Steps

### 1. Add `ReplyTool` to `agents/expert.py`
- New tool class that sends a message back to the requesting agent via bus
- The runner sets `_reply_to` and `_session_id` on the agent before calling `run()`
- `ReplyTool` reads these to know where to send

### 2. Add `SendMessageTool` to `agents/worker.py`
- New tool that sends a message to any agent or human via bus
- Replaces `DelegateToExpertTool` (remove it)
- Uses `_session_id` set by the runner

### 3. Update `agents/runner.py`
- Remove the `bus.send()` after `agent.run()` (lines 103-116)
- Set `_reply_to` (from_agent) and `_session_id` on the agent before calling `run()`
- Keep all logging as-is

### 4. Update system prompts
- Worker: instruct to use `send_message` for all outbound
- Expert (gmail): instruct to use `reply` for results

### 5. Wire `ReplyTool` into gmail expert factory (`experts/gmail.py`)
- Pass bus and register `ReplyTool` in the expert's tool registry

### 6. Update docs and README
