# Plan 0.0.0: Agentic Loop
**Date:** 2026-02-26
**Status:** Draft

## Context

Bootstrap pearscaff with a custom agentic loop built directly on the Anthropic SDK. The loop must support tool calling, be usable from the CLI (`pearscaff` / `ps`), and include two sample tools (math, web search).

## Pluggability Assessment

The spec asks whether the agentic loop and tooling should be pluggable (e.g. swap in LangChain later).

**Recommendation: Skip pluggability. Build direct.**

Reasons:
- many-bots used PydanticAI as the abstraction layer and it still ended up tightly coupled (message history, verbose streaming, graph nodes are all PydanticAI-specific). The abstraction bought little.
- A custom loop on the raw Anthropic SDK is already simple (~80 lines). Adding an abstraction layer doubles the code for a hypothetical future swap that may never happen.
- If a swap is ever needed, the tool definitions (the hard part) are already decoupled by design — they're just Python functions with type hints. The loop itself is the easy part to rewrite.
- LangChain adds massive dependency weight for what is fundamentally a `while` loop with tool dispatch.

**If you want pluggability anyway**, the minimal version would be: define a `Runner` protocol with `run(messages, tools) -> response` and implement `AnthropicRunner`. Tools stay the same either way. But I'd recommend against it at v0.0.0.

## Architecture

```
pearscaff/
├── cli.py              # Click CLI — `chat` command enters REPL
├── config.py           # Env config (ANTHROPIC_API_KEY, MODEL)
├── agent.py            # Agentic loop: send messages, handle tool_use, loop
├── tools/
│   ├── __init__.py     # Tool registry + BaseTool definition
│   ├── math.py         # MathTool — safe expression evaluator
│   └── web_search.py   # WebSearchTool — DuckDuckGo search
```

### Tool System

A tool is a class with:
- `name: str` — identifier passed to the Anthropic API
- `description: str` — what the LLM sees
- `input_schema: dict` — JSON Schema for parameters
- `execute(**kwargs) -> str` — runs the tool, returns text result

Registry: a simple dict `{name: tool_instance}` built at startup by scanning `tools/` or explicit registration.

### Agentic Loop (`agent.py`)

```
def run(user_message, messages, tools) -> str:
    append user_message to messages
    while True:
        response = client.messages.create(messages=messages, tools=tool_schemas)
        append assistant response to messages

        if stop_reason == "end_turn":
            return extract text from response

        if stop_reason == "tool_use":
            for each tool_use block in response:
                result = registry[tool_name].execute(**tool_input)
                append tool_result to messages
            continue  # loop back to LLM with results
```

That's it. No graph, no nodes, no framework. The Anthropic SDK handles message formatting, token counting, and streaming.

### CLI (`cli.py`)

Single command: `pearscaff chat` (or `ps chat`)
- Enters an interactive REPL
- Reads user input, passes to agent loop, prints response
- Ctrl+C / "exit" / "quit" to leave

## Implementation Steps

### 1. Add dependencies to `pyproject.toml`
- Add `anthropic` and `httpx` to dependencies

### 2. Update `config.py`
- Add `ANTHROPIC_API_KEY` (required, from env)
- Add `MODEL` with default `claude-sonnet-4-5-20250929`
- Add `MAX_TURNS` with default `10` (safety limit on loop iterations)

### 3. Create `pearscaff/tools/__init__.py`
- Define `BaseTool` class with `name`, `description`, `input_schema`, `execute()`
- Define `ToolRegistry` class: `register()`, `get()`, `all_schemas()`, `discover()`
- Auto-discovery: scan the `tools/` package for `BaseTool` subclasses

### 4. Create `pearscaff/tools/math.py`
- `MathTool` using `ast.parse()` for safe evaluation (no `eval()`)
- Supports: `+`, `-`, `*`, `/`, `//`, `**`, `%`, `sqrt`, `log`, `sin`, `cos`, `tan`, `abs`, `round`, `pi`, `e`

### 5. Create `pearscaff/tools/web_search.py`
- `WebSearchTool` hitting DuckDuckGo's API (`https://api.duckduckgo.com/`)
- Returns abstract text + top related topics
- Uses `httpx` for async HTTP

### 6. Create `pearscaff/agent.py`
- `Agent` class holding: Anthropic client, message history, tool registry, model config
- `run(user_message) -> str` implementing the loop above
- Safety: `MAX_TURNS` limit to prevent infinite tool loops
- Error handling: catch tool execution errors, return error text to LLM as tool_result

### 7. Update `pearscaff/cli.py`
- Add `chat` command with interactive REPL
- Initialize agent with config and discovered tools
- Print tool calls as they happen (name + args) for visibility
- Handle KeyboardInterrupt gracefully
