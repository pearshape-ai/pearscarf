# Plan 0.2.0: Experts & Workers Refactor
**Date:** 2026-02-27
**Status:** Draft

## Context

The current single Agent class becomes the "worker agent" (user-facing, Discord + CLI). A new "expert agent" type is introduced — domain-specialized agents that operate via CLI and accumulate knowledge. The first expert is a Gmail agent that uses a headless browser to operate Gmail's web UI.

The knowledge storage pattern is borrowed from the workers-experts repo.

### Agent Hierarchy

```
BaseAgent (agents/base.py)
  - Anthropic client, message history, agentic loop
  - on_tool_call callback
  - Extracted from current Agent.run()

WorkerAgent(BaseAgent) (agents/worker.py)
  - Same as current Agent, uses tool registry
  - Used by CLI chat + Discord

ExpertAgent(BaseAgent) (agents/expert.py)
  - Has a KnowledgeStore reference
  - System prompt includes loaded knowledge from its domain folder
  - Domain-specific tools
  - Stores learnings about how to operate its domain (selectors, navigation patterns, gotchas)
```

### Gmail Expert — Headless Browser

Uses **Playwright** to operate Gmail's web UI in a headless browser. The agent controls the browser through tools to log in, navigate, read emails, and mark as read.

**Tools (registered for the expert's agentic loop):**
- `browser_navigate` — go to a URL
- `browser_click` — click an element by selector
- `browser_type` — type text into an element
- `browser_get_text` — extract text content from elements
- `browser_screenshot` — take a screenshot (returns description of what's visible)
- `browser_wait` — wait for a selector to appear
- `gmail_get_unread` — navigate to inbox, extract unread email list (subjects, senders)
- `gmail_read_latest` — open the latest unread email, extract full content
- `gmail_mark_as_read` — mark the currently open email as read

The lower-level browser tools give the agent flexibility to navigate Gmail even when the UI changes. The higher-level gmail_ tools encode known-good workflows.

**Auth flow:**
- Playwright uses a persistent browser context (`storage_state.json`) so the user logs in once manually (agent opens visible browser, user enters credentials + 2FA), then subsequent runs reuse the saved session.
- `storage_state.json` is gitignored.

**Knowledge collection:**
The expert stores knowledge about how to effectively operate the Gmail web app:
- CSS selectors that work for finding inbox items, email content, buttons
- Navigation patterns (how to get to inbox, how to open an email, how to mark as read)
- Gotchas and workarounds (elements that take time to load, dynamic class names to avoid)
- Anything it learns that helps it reliably access Gmail over and over

This knowledge is saved as markdown in `knowledge/gmail/` and loaded into the expert's system prompt on startup, so it gets better with use.

**Output:**
- Emails are printed to the terminal (subject, sender, date, body summary). Nothing is saved.
- Detailed reasoning, tool calls and other relevant execution info are also printed on the terminal as the expert agent executes.

### CLI Commands

```
pearscaff chat              # WorkerAgent REPL (unchanged)
pearscaff discord           # WorkerAgent Discord bot (unchanged)
pearscaff expert gmail      # GmailExpert REPL
pearscaff expert gmail --login   # Opens visible browser for initial Gmail login
```

## Implementation Steps

### 1. Add dependencies
- `playwright` to pyproject.toml
- Run `playwright install chromium` post-install

### 2. Create `pearscaff/knowledge/__init__.py`
- `KnowledgeStore` class
- `save(domain, name, content)` — writes markdown to `knowledge/<domain>/<name>.md`
- `load_all(domain) -> str` — reads all markdown in a domain folder, concatenates
- `list(domain) -> list[str]` — list stored knowledge files

### 3. Create `pearscaff/agents/` package
- `agents/base.py` — extract `BaseAgent` from current `agent.py` with the agentic loop
- `agents/worker.py` — `WorkerAgent(BaseAgent)` (same behavior as current)
- `agents/expert.py` — `ExpertAgent(BaseAgent)` with `KnowledgeStore`, domain system prompt that includes loaded knowledge, and a `save_knowledge(name, content)` tool built-in

### 4. Create `pearscaff/experts/gmail.py`
- Browser management: `launch_browser()`, `get_page()`, persistent context with `storage_state.json`
- Browser tools as `BaseTool` subclasses: navigate, click, type, get_text, screenshot, wait
- Gmail-specific tools: `get_unread`, `read_latest`, `mark_as_read`
- `GmailExpert` class: wires up ExpertAgent with browser tools + knowledge store
- `--login` mode: launches headed browser, waits for user to log in, saves storage state

### 5. Create `pearscaff/experts/__init__.py`
- Expert registry: `{"gmail": GmailExpert}`

### 6. Update CLI
- Add `expert` command group with `gmail` subcommand
- `--login` flag for initial auth
- REPL for expert interaction — user asks, expert operates browser, prints results to terminal

### 7. Update imports
- `discord_bot.py` → import from `agents.worker`
- `cli.py` `chat` command → import from `agents.worker`
- Delete old `agent.py`

### 8. Update gitignore
- Add `storage_state.json`, `knowledge/gmail/*.md` (knowledge is local, not committed)
