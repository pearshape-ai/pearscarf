```sql
CREATE TABLE IF NOT EXISTS issues (
    record_id TEXT PRIMARY KEY REFERENCES records(id),
    linear_id TEXT UNIQUE NOT NULL,
    identifier TEXT,
    title TEXT NOT NULL,
    status TEXT,
    priority TEXT,
    assignee TEXT,
    project TEXT,
    labels JSONB,
    url TEXT,
    linear_created_at TIMESTAMPTZ,
    linear_updated_at TIMESTAMPTZ
);

CREATE INDEX IF NOT EXISTS idx_issues_identifier ON issues(identifier);
```

## Step 5 тАФ Create Linear expert agent: `pearscaff/experts/linear.py`

Following the Gmail expert pattern.

### Tools

Each tool wraps a `LinearClient` method:

**`LinearListIssuesTool`** тАФ list/filter issues
- Input: `status`, `assignee`, `project`, `priority`, `label` (all optional)
- Calls `client.list_issues(...)`, formats results

**`LinearGetIssueTool`** тАФ get issue by identifier
- Input: `identifier` (e.g. "ENG-42")
- Calls `client.get_issue(identifier)`, formats full details

**`LinearCreateIssueTool`** тАФ create an issue
- Input: `title`, `description`, `team`, `assignee`, `priority`, `project`, `labels`
- Resolves names to IDs via `list_teams`/`list_users`/`list_projects`/`list_labels` (cached)
- Calls `client.create_issue(...)`, returns result

**`LinearUpdateIssueTool`** тАФ update an issue
- Input: `identifier`, `status`, `priority`, `assignee`, `labels`
- Fetches issue to get `issue_id`, then calls `client.update_issue(...)`

**`LinearAddCommentTool`** тАФ add a comment
- Input: `identifier`, `body`
- Fetches issue to get `issue_id`, then calls `client.add_comment(...)`

**`LinearSearchIssuesTool`** тАФ text search
- Input: `query`
- Calls `client.search_issues(query)`, formats results

**`SaveIssueTool`** тАФ save issue to SOR (same pattern as `SaveEmailTool`)
- Input: issue fields
- Calls `store.save_issue(...)`

### Factory functions

**`create_linear_expert_for_runner(bus)`** тАФ returns `(factory, client_or_none)`
- If `LINEAR_API_KEY` is set, creates a `LinearClient` and returns a factory that builds `ExpertAgent` with Linear tools
- If not set, returns `None` (Linear expert not available)

**`start_issue_polling(bus, client, interval)`** тАФ same pattern as `start_email_polling`
- Background daemon thread
- First run: load active issues, save to SOR
- Subsequent: query issues updated since last sync
- New issues тЖТ create session тЖТ send to worker for triage
- Stores `synced_at` cursor in memory (thread-local)

**File:** `pearscaff/experts/linear.py` (new)
