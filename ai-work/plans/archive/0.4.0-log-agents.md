# Plan 0.4.0: Agent Logging
**Date:** 2026-02-27
**Status:** Draft

## Context

Agents communicate via SQLite but there's no unified observability into what they're doing. This version adds a single shared log file (`logs/session.log`) that reads as a full timeline of every agent action, tool call, thought, and message — across all agents.

## Architecture

```
pearscaff/
├── log.py                 # NEW: shared logger, write() function, format helpers
├── agents/
│   ├── base.py            # MODIFIED: emit log entries for tool calls, text, errors
│   └── runner.py          # MODIFIED: emit log entries for message_sent, message_received, polling
├── agents/worker.py       # MODIFIED: wire logger into agent factory callbacks
├── experts/gmail.py       # MODIFIED: wire logger into expert factory callbacks
├── repl.py                # MODIFIED: log human input
└── discord_bot.py         # MODIFIED: log human input from Discord
```

Only one new file: `log.py`. Everything else is wiring the existing callback hooks (`on_tool_call`, `on_text`, `on_tool_result`) to also write log entries, plus adding `message_sent`/`message_received` logging in the runner and bus.

## Log Format

```
[2026-02-27T10:30:01Z] [gmail_expert] [ses_001] [tool] gmail_get_unread({})
[2026-02-27T10:30:02Z] [gmail_expert] [ses_001] [result] gmail_get_unread: found 3 unread emails
[2026-02-27T10:30:02Z] [gmail_expert] [ses_001] [thinking] 3 unread emails found...
[2026-02-27T10:30:03Z] [worker] [--] [action] polling for unread messages, none found
```

## `log.py` Design

Single module with a file-backed append-only logger:

```python
_log_file: Path = Path("logs/session.log")

def write(agent: str, session: str | None, entry_type: str, message: str) -> None:
    # Appends one line: [timestamp] [agent] [session or --] [entry_type] message
```

- Thread-safe (uses a lock)
- Creates `logs/` dir on first write
- Append-only, never truncates
- Entry types: `action`, `message_sent`, `message_received`, `reasoning`, `thinking`, `tool`, `result`, `error`

## Integration Points

### BaseAgent (`agents/base.py`)
The agent doesn't know its own name or session ID. Instead, the existing callbacks (`on_tool_call`, `on_text`, `on_tool_result`) are already the integration point. The caller (runner, CLI) wires these to both the terminal output and the logger.

### AgentRunner (`agents/runner.py`)
The runner knows the agent name and session ID. It:
- Logs `[message_received]` when it picks up a message
- Logs `[message_sent]` when it sends a response
- Logs `[error]` on exceptions
- Logs `[action]` on empty polls (optional, low frequency)
- Wires `on_tool_call`, `on_text`, `on_tool_result` callbacks on the agent factory to emit log entries

### Worker factory (`agents/worker.py`)
`create_worker_agent()` already accepts callbacks. The runner will pass callbacks that log.

### Gmail expert factory (`experts/gmail.py`)
`create_gmail_expert_for_runner()` factory returns agents without callbacks. Updated to accept callbacks from the runner.

### REPL (`repl.py`)
Logs `[message_sent]` when human sends a message, `[message_received]` when human gets a response.

### Discord bot (`discord_bot.py`)
Same as REPL: log human messages and responses.

## Implementation Steps

### 1. Create `pearscaff/log.py`
- `write(agent, session, entry_type, message)` function
- Thread-safe with `threading.Lock`
- Auto-creates `logs/` directory
- Add `logs/` to `.gitignore`

### 2. Update `agents/runner.py`
- Import `log.write`
- Log `message_received` in `_process_message`
- Log `message_sent` after sending response
- Log `error` on exceptions
- Wire `on_tool_call`, `on_text`, `on_tool_result` into agent factories to emit log entries with correct agent name and session ID

### 3. Update `agents/worker.py`
- Update `create_worker_agent()` to accept and merge external callbacks with logging callbacks

### 4. Update `experts/gmail.py`
- Update `create_gmail_expert_for_runner()` factory to return agents with logging callbacks

### 5. Update `repl.py`
- Log when human sends a message
- Log when human receives a response

### 6. Update `discord_bot.py`
- Log when human sends a message from Discord
- Log when response is posted to a thread

### 7. Update `.gitignore`
- Add `logs/`
