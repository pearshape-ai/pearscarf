# Plan 0.4.6: Fix Startup Message Spacing
**Date:** 2026-03-01
**Status:** Draft

## Context

Startup messages ("Gmail expert started.", "Worker agent started.", "PearScaff v0.4.5...") render with the same horizontal drift as the response text bug from v0.4.5. Each line starts where the previous ended instead of at column 0.

## Root Cause

Two issues:

### 1. `_original_termios` captures broken state

`terminal.py` saves `_original_termios = termios.tcgetattr(...)` at import time. If the terminal is already in raw mode from a previous crashed session, this captures the RAW mode settings. Then `_restore_terminal()` in `cli.py` "restores" to raw mode — making things worse, not better.

### 2. Startup messages bypass `_write()`

Startup messages use `click.echo()` which writes directly to stdout without the `\n` → `\r\n` conversion that `TerminalUI._write()` does. If the terminal is in raw mode (due to issue 1), these messages get the column drift.

## Fix

### `terminal.py`

1. Remove the module-level `_original_termios` capture
2. Save known-good terminal state inside `read_line()` BEFORE entering raw mode — at that point we know the terminal is in cooked mode
3. The `atexit`/`SIGTERM` handlers use this saved state

### `repl.py` + `cli.py`

Route startup messages through `TerminalUI._write()` so they get the `\r\n` conversion. Add a `println()` convenience method to TerminalUI for simple line output.

## Implementation Steps

### 1. Fix terminal state capture in `terminal.py`
- Remove module-level `_original_termios = termios.tcgetattr(...)`
- Add `println(text)` method to TerminalUI — simple `_write(text + "\n")`
- In `read_line()`, before `tty.setraw()`, save settings to module-level `_original_termios` for the atexit/signal handlers

### 2. Update `repl.py` startup
- Replace `click.echo()` startup messages with `self._ui.println()`

### 3. Update `cli.py` startup
- Create a TerminalUI early and use `println()` for "Gmail expert started." etc.
- OR just use `sys.stdout.write("...\r\n")` directly since the TerminalUI is created inside SessionRepl

### 4. Bump version to 0.4.6
