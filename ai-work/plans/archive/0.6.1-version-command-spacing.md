# Plan 0.6.1: Fix Version Command Output
**Date:** 2026-03-02
**Status:** Executed

## Context

`ps --version` outputs `PearScaff, version 0.6.0` followed by a stray `%` character at an indented position. The `%` is zsh's `PROMPT_EOL_MARK`. The format also doesn't match the desired `PearScaff v0.6.1` style.

## Root Cause

Two issues stacking:

1. **Format**: Click's `@click.version_option` defaults to `"%(prog)s, version %(version)s"`. Needed `PearScaff v0.6.1`.

2. **Stale raw mode**: The terminal was left in raw mode from a previous crashed `pearscaff run` session. In raw mode, `\n` moves the cursor down but doesn't return to column 0. So after printing `PearScaff v0.6.1\n`, the cursor lands at column 16 on the next line. Zsh's `%` marker appears there instead of at column 0.

The initial fix (changing `message` format) didn't help because the `%` wasn't a format issue — it was a terminal state issue. The second attempt (explicit `sys.stdout.write` + flush) also didn't help for the same reason.

## Fix

### `stty sane` at CLI import time

Added `os.system("stty sane 2>/dev/null")` at module level in `cli.py`. This runs before any command or flag processing, resetting the terminal to cooked mode regardless of what a previous crashed session left behind.

### Replace `click.version_option` with manual callback

Replaced Click's built-in `@click.version_option` with a custom `--version` flag using `click.option` + `is_eager=True` callback. The callback uses `click.echo()` for output — which works correctly now that `stty sane` has already run.

### Remove redundant `_restore_terminal()` from `run` command

The `_restore_terminal()` call and `terminal.py` import in the `run` command were no longer needed since `stty sane` handles it at import time.

## Changes Made

### `cli.py`
- Added `import os` and `os.system("stty sane 2>/dev/null")` at module level
- Replaced `@click.version_option(...)` with custom `_print_version` callback via `@click.option("--version", ...)`
- Removed `from pearscaff.terminal import _restore_terminal` and `_restore_terminal()` from `run` command

### `__init__.py`
- Bumped to `0.6.1`

## Verification

```bash
# Even after a crashed pearscaff run that left terminal in raw mode:
ps --version
# → PearScaff v0.6.1
# No stray % character, output at column 0
```
