# Plan 0.9.3: Discord Bot Fixes
**Date:** 2026-03-03
**Status:** Executed

## Context

Two issues with the Discord bot after recent changes:

1. **Role mention not recognized** — Discord auto-creates a managed role named "ps" for the bot. When users type `@ps`, Discord often resolves it to the role mention (`<@&ROLE_ID>`) instead of the user mention (`<@USER_ID>`). The bot only checked for user mentions via `self.user.mentioned_in(message)`, so role mentions were silently dropped — `on_message` returned early and messages were never processed.

2. **Silent failures in on_message** — The `on_message` handler had no error handling. Any exception (rate limit, DB lock, etc.) would silently drop the message with no log entry and no feedback to the user.

## Root Cause

### Role mention
Discord's `mentioned_in()` only checks user mentions (`<@ID>`), not role mentions (`<@&ID>`). The bot's managed role shares its name, so `@ps` in Discord resolves to the role. The code treated this as "not mentioned" and returned.

### Silent failures
The `on_message` async handler had no try/except. Discord.py catches unhandled exceptions internally and continues, but the message is lost with no trace.

## Fix

### `discord_bot.py`

1. **Role mention detection** — Check both `self.user.mentioned_in(message)` (user mention) and `message.role_mentions` for any role matching the bot's name. Strip both mention formats from message content.

2. **Error handling** — Wrapped the message processing logic in `_handle_message()` called from `on_message` inside a try/except. Errors are printed to terminal, logged to session.log, and replied to the user on Discord.

## Changes Made

### `discord_bot.py`
- Added `is_role_mention` check: matches `message.role_mentions` against `self.user.name`
- `is_mention` is now `is_user_mention or is_role_mention`
- Strips both `<@BOT_ID>` and `<@&ROLE_ID>` from message content
- Extracted message processing into `_handle_message()` method
- `on_message` wraps `_handle_message()` in try/except with logging and Discord error reply
