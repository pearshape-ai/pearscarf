# Plan: 1.5.0 — Wire Qdrant Vector Search

## Summary

Un-stub `vectorstore.py` to embed and search email content in Qdrant. Wire the Indexer to embed after extraction, the Retriever's `VectorSearchTool` to query Qdrant, and the memory CLI's search/list commands to use semantic search.

---

## Step 1 — Un-stub `vectorstore.add_record`

Replace the stubbed body:

- Embed `content` via `_embed()`
- Build payload: `{"record_id": record_id, "content": content[:1000], ...metadata}` (sender, subject, type, source)
- Upsert to Qdrant using `_record_id_to_uuid(record_id)` as the point ID

Content stored in payload should be truncated for storage (first ~1000 chars) — enough for search result display without a second lookup.

**File:** `pearscaff/vectorstore.py`

---

## Step 2 — Un-stub `vectorstore.query`

Replace the stubbed body:

- Embed `query_text` via `_embed()`
- Search Qdrant with `search()` using the embedded vector, limit `n_results`
- Return list of dicts: `{"id": record_id, "content": ..., "metadata": ..., "score": distance}`

**File:** `pearscaff/vectorstore.py`

---

## Step 3 — Wire Indexer to embed after extraction

In `_process_record`, after writing to Neo4j and before `_mark_indexed`:

- Import `vectorstore`
- Call `vectorstore.add_record(record_id, content, metadata)` where metadata includes record type, source, and email fields (sender, subject)
- Build metadata from the email record (same info available from `_build_content`'s DB query)

Wrap in try/except so a Qdrant failure doesn't block indexing.

**File:** `pearscaff/indexer.py`
