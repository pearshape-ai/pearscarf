# Plan 0.4.2: Versioning
**Date:** 2026-02-28
**Status:** Draft

## Context

There's no way to verify which version of PearScaff is running. The version in `pyproject.toml` is stale at `0.1.0`. This version adds a single source of truth for the version string, a `--version` CLI flag, and prints the version in the REPL on startup.

## Architecture

```
pearscaff/
├── __init__.py            # MODIFIED: add __version__ = "0.4.2"
├── cli.py                 # MODIFIED: add --version flag to cli group
├── repl.py                # MODIFIED: print version on startup
pyproject.toml             # MODIFIED: update version to 0.4.2, read from package
```

The single source of truth is `pearscaff/__init__.py` with `__version__ = "0.4.2"`. `pyproject.toml` uses hatchling's dynamic versioning to read from the package, keeping them in sync.

## Implementation Steps

### 1. Set `__version__` in `pearscaff/__init__.py`
- Add `__version__ = "0.4.2"` to the package init

### 2. Update `pyproject.toml`
- Change `version = "0.1.0"` to `dynamic = ["version"]`
- Add `[tool.hatch.version]` section pointing to `pearscaff/__init__.py`

### 3. Add `--version` to CLI (`cli.py`)
- Use Click's built-in `@click.version_option` on the cli group
- Import `__version__` from `pearscaff`
- Running `ps --version` or `pearscaff --version` prints `PearScaff v0.4.2`

### 4. Print version in REPL startup (`repl.py`)
- Import `__version__` from `pearscaff`
- Change the startup message from `"pearscaff (type 'exit' or Ctrl+C to quit)"` to `"PearScaff v0.4.2 (type 'exit' or Ctrl+C to quit)"`

### 5. Update docs and README

## Verification

```bash
uv sync

# CLI version flag
pearscaff --version
# → PearScaff, version 0.4.2

ps --version
# → PearScaff, version 0.4.2

# → PearScaff v0.4.2 (type 'exit' or Ctrl+C to quit)
# → [ses_001] >
```
