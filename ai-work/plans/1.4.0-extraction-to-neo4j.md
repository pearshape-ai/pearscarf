## Step 3 — Rewrite `graph.py` to use Neo4j

Replace all stubbed function bodies with Neo4j Cypher queries. Keep the same function signatures.

### Entity resolution (`find_entity`)
- `MATCH (n:{Label}) WHERE n.name = $name RETURN n`
- For persons: also match on `n.email` if provided
- For companies: also match on `n.domain` if provided

### Entity creation (`create_entity`)
- `MERGE` on name+label (plus email for person, domain for company)
- `ON CREATE SET` all metadata properties + `created_at`
- `ON MATCH SET` new metadata properties (additive — don't overwrite existing)
- Return the element ID as the entity identifier

### Entity lookup (`get_entity`)
- `MATCH (n) WHERE elementId(n) = $id RETURN n`

### Entity search (`search_entities`)
- `MATCH (n) WHERE toLower(n.name) CONTAINS toLower($query) RETURN n LIMIT $limit`
- Optionally filter by label if `entity_type` provided
- Also search `n.email` and `n.domain` properties

### Edge creation (`create_edge`)
- `MATCH (a), (b) WHERE elementId(a) = $from AND elementId(b) = $to`
- `CREATE (a)-[r:$type {source_record: $source, created_at: $ts}]->(b)`
- Dynamic relationship type from extraction (WORKS_AT, CUSTOMER_OF, etc.)

### Fact storage (`upsert_fact`)
- Facts become `Fact` nodes connected to the entity: `(entity)-[:HAS_FACT]->(f:Fact)`
- Fact node properties: `claim`, `confidence`, `source_record`, `created_at`
- MERGE on entity + claim text to avoid duplicates

### Fact retrieval (`get_entity_facts`)
- `MATCH (n)-[:HAS_FACT]->(f:Fact) WHERE elementId(n) = $id RETURN f`

### Graph traversal (`traverse_graph`)
- Variable-length path query up to `max_depth` hops
- Return connected entities, relationship types, and source records

### `list_entity_types` — unchanged (still reads from Postgres)

**File:** `pearscaff/graph.py`

## Step 4 — Un-stub the Indexer: wire extraction + Neo4j writes

### `_extract` — call Claude API
- Load system prompt via `load_prompt("extraction")`
- Build user message: `f"Record ({record_id}, {record_type}):\n\n{content}"`
- Call `anthropic.Anthropic().messages.create()` with `EXTRACTION_MODEL`, `EXTRACTION_MAX_TOKENS`, `EXTRACTION_TEMPERATURE`, system/user split
- Parse the JSON response (reuse `_parse_json_response` pattern from `extract_test.py`)
- Return the parsed dict with `entities`, `relationships`, `facts`

### `_resolve_entity` — entity resolution via graph.py
- For each extracted entity: call `graph.find_entity(type, name, metadata)`
- If found, return the existing entity ID
- If not found, call `graph.create_entity(type, name, metadata)` and return the new ID

### `_process_record` — full pipeline
1. `_build_content(record)` (already works)
2. `extracted = _extract(record_type, record_id, content)`
3. If extraction returned empty/failed, log and mark indexed
4. For each entity in `extracted["entities"]`: resolve → get entity ID
5. For each relationship in `extracted["relationships"]`: look up from/to entity IDs by name, call `graph.create_edge`
6. For each fact in `extracted["facts"]`: look up entity ID by name, call `graph.upsert_fact`
7. `_mark_indexed(record_id)`

### New imports
- `anthropic`, `json`
- `pearscaff.config` (EXTRACTION_MODEL, EXTRACTION_MAX_TOKENS, EXTRACTION_TEMPERATURE, ANTHROPIC_API_KEY)
- `pearscaff.prompts.load`
- `pearscaff.graph`

**File:** `pearscaff/indexer.py`
