# Plan: 1.3.0 — Decouple Prompts from Code

Extract all system prompts from Python string constants into standalone markdown files under `pearscaff/prompts/`. Add a simple loader utility. No prompt content changes.

## Steps

### Step 1: Create `pearscaff/prompts/` folder and prompt files

Create the folder and extract these prompts as-is (no content changes):

| File | Source | Constant |
|---|---|---|
| `worker.md` | `agents/worker.py` | `WORKER_SYSTEM_PROMPT` |
| `gmail_browser.md` | `experts/gmail.py` | `GMAIL_SYSTEM_PROMPT_BROWSER` |
| `gmail_mcp.md` | `experts/gmail.py` | `GMAIL_SYSTEM_PROMPT_MCP` |

| `extraction.md` | `indexer.py` | `EXTRACTION_TEMPLATE` (commented out — uncomment into the file) |

Each file is plain markdown — the file content IS the prompt. No frontmatter, no wrappers.

The extraction template keeps its Python format-string placeholders (`{record_type}`, `{entity_types_block}`, `{record_id}`, `{content}`) and double-brace escapes (`{{`, `}}`). The calling code will `.format()` them as before.

### Step 2: Create `pearscaff/prompts/__init__.py` — prompt loader

A single function:

```python
def load(name: str) -> str:
    """Load a prompt by name from the prompts folder. Returns the file content as a string."""
```

- Takes a name like `"worker"` or `"gmail_mcp"`
- Reads `pearscaff/prompts/{name}.md`
- Returns the content as a string
- Uses `importlib.resources` (or `Path(__file__).parent / f"{name}.md"`) to locate the file relative to the package
- Raises `FileNotFoundError` if the prompt file doesn't exist

No caching, no hot-reload, no validation.
