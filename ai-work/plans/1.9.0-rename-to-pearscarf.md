# Plan: 1.9.0 — Rename PearScaff to PearScarf

Full rename across the entire codebase. No functional changes — purely a name swap.

## Execution Strategy

The rename has two phases:
1. **Directory rename**: `mv pearscaff/ pearscarf/` — must happen first
2. **Global find-and-replace**: `pearscaff` → `pearscarf` and `PearScaff` → `PearScarf` across all files

Phase 2 is a bulk operation — nearly every `.py` file, all docs, config files, and docker-compose. Rather than listing per-file changes, the plan describes the rename rules and then the special cases.

## Phase 1: Directory Rename

```bash
mv pearscaff pearscarf
```

This renames the Python package directory. All subsequent file paths reference `pearscarf/`.

## Phase 2: Global Find-and-Replace

Apply these substitutions across all tracked files (excluding `.venv/`, `data/`, `.git/`):

| Pattern | Replacement | Scope |
|---------|-------------|-------|
| `pearscaff` (lowercase, in imports/code) | `pearscarf` | All `.py` files, `pyproject.toml` |
| `PearScaff` (title case, in prose) | `PearScarf` | All `.md` files, docstrings, comments |
| `Pearscaff` (class name) | `Pearscarf` | `discord_bot.py` (`PearscaffBot` → `PearscarfBot`) |
| `PEARSCAFF` (if any uppercase) | `PEARSCARF` | Check but unlikely |

## Special Cases

### 1. Postgres defaults in config.py

```python
POSTGRES_USER = os.getenv("POSTGRES_USER", "pearscarf")
POSTGRES_DB = os.getenv("POSTGRES_DB", "pearscarf")
```

**Important**: The running Docker Postgres already has user `pearscaff` and database `pearscaff`. Changing the defaults means the user needs to either:
- Update their `.env` file to `POSTGRES_USER=pearscarf` / `POSTGRES_DB=pearscarf` and recreate the DB, OR
- Keep their `.env` pointing to `pearscaff` (old name) since the defaults only matter when env vars aren't set

The `.env` file should be updated to the new name. The user will need to recreate the Postgres container with the new database name. Add a note in the CHANGELOG.

### 2. `.env` file (actual, not just example)

The actual `.env` file has `POSTGRES_USER=pearscaff`, `POSTGRES_DB=pearscaff`, `PGADMIN_EMAIL=admin@pearscaff.dev`. These should be updated too, but since `.env` is gitignored, document the needed changes rather than modifying it directly.

### 3. Discord bot class name

`PearscaffBot` → `PearscarfBot` in `discord_bot.py`. This is an internal class name, not user-visible.

### 4. CLI entry point

The `pearscaff` CLI command becomes `pearscarf`. The `ps` alias stays as-is. After rename, user needs to `pip install -e .` again to register the new entry points.

### 5. User-facing error messages

In `cli.py` and `gmail.py`, error messages reference `pearscaff` commands:
- `"Run 'pearscaff gmail --auth' to set up OAuth."` → `"Run 'pearscarf gmail --auth'..."`
- `"Run 'pearscaff expert gmail --login' to re-authenticate."` → `"Run 'pearscarf expert gmail --login'..."`

These get caught by the global replace.

### 6. LangSmith project name

Currently `"pears"` — not `"pearscaff"`. No change needed.

### 7. CHANGELOG

Add a 1.9.0 entry noting the rename. Existing entries reference "pearscaff" but that's historical — leave them as-is since they describe what happened at that point in time.
