## Steps

### Step 1: Rewrite `pearscaff/vectorstore.py` — Qdrant backend

**File:** `pearscaff/vectorstore.py`

Replace the ChromaDB wrapper with a Qdrant-backed implementation. Keep the same public interface: `add_record()`, `query()`.

Key changes:
- Import `qdrant_client` and `sentence_transformers` directly
- Lazy-init `QdrantClient` connecting to `QDRANT_URL` (default `http://localhost:6333`)
- Lazy-init `SentenceTransformer("all-MiniLM-L6-v2")` for embedding
- Collection name: `records` (same as ChromaDB)
- `add_record(record_id, content, metadata)` — embed text, upsert point with payload
- `query(query_text, n_results, where)` — embed query, search, return same dict format `{id, content, metadata, distance}`
- Auto-create collection on first use if it doesn't exist (vector size 384 for MiniLM-L6-v2, cosine distance)
- Metadata filtering: translate the `where` dict to Qdrant filter conditions

```python
"""Vector storage layer — Qdrant wrapper.

Lazy-initialized. The Qdrant client and sentence-transformers model
only load on first use, so commands that don't need vector search stay fast.
"""

from __future__ import annotations

from typing import Any

from pearscaff.config import QDRANT_URL

_client = None
_model = None

COLLECTION_NAME = "records"
VECTOR_SIZE = 384  # all-MiniLM-L6-v2 output dimension


def _get_client():
    global _client
    if _client is None:
        from qdrant_client import QdrantClient
        _client = QdrantClient(url=QDRANT_URL)
        _ensure_collection()
    return _client


def _get_model():
    global _model
    if _model is None:
        from sentence_transformers import SentenceTransformer
        _model = SentenceTransformer("all-MiniLM-L6-v2")
    return _model


def _ensure_collection():
    from qdrant_client.models import Distance, VectorParams
    collections = [c.name for c in _client.get_collections().collections]
    if COLLECTION_NAME not in collections:
        _client.create_collection(
            collection_name=COLLECTION_NAME,
            vectors_config=VectorParams(size=VECTOR_SIZE, distance=Distance.COSINE),
        )


def _embed(text: str) -> list[float]:
    model = _get_model()
    return model.encode(text).tolist()


def add_record(record_id: str, content: str, metadata: dict) -> None:
    from qdrant_client.models import PointStruct
    client = _get_client()
    vector = _embed(content)
    payload = {**metadata, "content": content}
    client.upsert(
        collection_name=COLLECTION_NAME,
        points=[PointStruct(id=record_id, vector=vector, payload=payload)],
    )


def query(
    query_text: str,
    n_results: int = 5,
    where: dict | None = None,
) -> list[dict]:
    from qdrant_client.models import FieldCondition, Filter, MatchValue
    client = _get_client()
    vector = _embed(query_text)

    query_filter = None
    if where:
        conditions = []
        for key, value in where.items():
            conditions.append(FieldCondition(key=key, match=MatchValue(value=value)))
        query_filter = Filter(must=conditions)

    results = client.search(
        collection_name=COLLECTION_NAME,
        query_vector=vector,
        limit=n_results,
        query_filter=query_filter,
    )

    output = []
    for hit in results:
        payload = hit.payload or {}
        output.append({
            "id": hit.id,
            "content": payload.get("content", ""),
            "metadata": {k: v for k, v in payload.items() if k != "content"},
            "distance": 1.0 - hit.score,  # Qdrant cosine returns similarity, callers expect distance
        })
    return output
```

**Note on IDs:** Qdrant requires point IDs to be integers or UUIDs. Record IDs in PearScaff are strings like `email_001`. Two options:
- Use a deterministic UUID derived from the string ID (e.g. `uuid5(NAMESPACE_URL, record_id)`)
- Store the string ID in the payload and use a hash as the point ID

Recommended: use `uuid5` for clean bidirectional mapping.

Update `_embed` and `add_record`/`query` to use `uuid.uuid5(uuid.NAMESPACE_URL, record_id)` as the Qdrant point ID, and store the original string ID in the payload for retrieval.

## Notes

- **Qdrant point IDs**: PearScaff uses string IDs (`email_001`). Qdrant requires int or UUID. Plan uses `uuid5(NAMESPACE_URL, record_id)` for deterministic mapping. The original string ID is stored in the payload.
- **Distance vs similarity**: Qdrant cosine search returns similarity (0→1, higher=better). ChromaDB returned distance (0→2, lower=better). The new vectorstore converts: `distance = 1.0 - similarity`. Callers already sort by distance ascending, so this preserves behavior.
- **Existing data**: ChromaDB data in `data/chroma/` becomes orphaned. Qdrant collection starts empty. Records need re-indexing: `UPDATE records SET indexed = 0 WHERE classification = 'relevant'`.
- **Qdrant Docker**: Already running from Mem0 setup. May contain old Mem0 data in a `mem0` collection — that's separate from the new `records` collection.
