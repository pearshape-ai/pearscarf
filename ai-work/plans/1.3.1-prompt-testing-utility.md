# Plan: 1.3.1 — Prompt Testing Utility

A standalone script for iterating on the extraction prompt. Reads emails from Postgres, runs them through the extraction prompt via Claude, prints results. No writes to any storage.

## Steps

### Step 1: Create `scripts/test_extraction.py`

A CLI script using `dotenv` + direct Anthropic SDK:

```
Usage:
    python scripts/test_extraction.py                  # all relevant emails
    python scripts/test_extraction.py email_042         # single record
    python scripts/test_extraction.py email_001 email_005  # multiple records
```

**Logic:**

1. Load `.env` via `dotenv`
2. Call `init_db()` to ensure Postgres schema is ready
3. If record IDs provided as args, use those. Otherwise query all records with `classification = 'relevant'`, ordered by `created_at`.
4. For each record:
   a. Load the record from `records` table (get `id`, `type`, `source`, `created_at`, `raw`, `human_context`)
   b. Build content using `Indexer._build_content` logic — but since that's an instance method, extract the content-building inline or reuse the same SQL query from indexer (read from `emails` table for email type, fallback to `raw`). Simplest: just instantiate `Indexer` and call `_build_content(record)`, or duplicate the 10 lines of SQL inline. Prefer reuse — instantiate the Indexer just for `_build_content` and `_build_entity_types_block`.
   c. Load extraction prompt via `load_prompt("extraction")`
   d. Format the prompt with `record_type`, `entity_types_block`, `record_id`, `content`
   e. Append human_context if present
   f. Call Claude via `anthropic.Anthropic().messages.create()`
   g. Parse JSON from response (handle ```json fencing)
   h. Print: separator, email content, separator, raw JSON extraction, token usage

5. If LangSmith tracing is enabled, wrap the LLM call in `trace_span` from `pearscaff.tracing`. Low effort — just a `with trace_span(...)` around the API call.

**Output format per record:**

```
═══════════════════════════════════════════════════════
email_042
═══════════════════════════════════════════════════════

--- Email Content ---
From: john@acme.com
To: me@example.com
Subject: Q3 Invoice
Date: 2026-03-10
...

--- Extraction Result ---
{
  "entities": [...],
  "relationships": [...],
  "facts": [...]
}

(tokens: 450 in / 120 out)
```

If JSON parsing fails, print the raw text response with an error note instead of crashing.

### Step 2: Add CLI command `pearscaff extract-test`

Wire the script as a Click command in `cli.py` so it's also accessible via:

```bash
pearscaff extract-test                    # all relevant
pearscaff extract-test email_042          # single
pearscaff extract-test email_001 email_005  # multiple
```

This is a thin wrapper that calls the same logic as the script. Keeps both entry points (script for quick runs, CLI for consistency).
