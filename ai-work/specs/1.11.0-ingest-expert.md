# Ingest expert scaffolding

## Context

- PearScarf currently has three expert agents: gmail, linear, retriever
- No mechanism exists to load data from files into the system
- The ingest expert is a new expert agent that owns file-based data entry
- Two ingestion modes are needed: seed files (typed block markdown format) and typed JSON record files
- This spec establishes the expert, its prompt, its tools, and the CLI subcommand
- Storage wiring and indexer integration are intentionally excluded — this is scaffolding only
- The ingest expert follows the same architectural pattern as gmail and linear experts
