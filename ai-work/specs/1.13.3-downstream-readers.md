# Downstream readers updated to new fact model field names

## Context

- `graph.py` now writes `edge_label`, `fact_type`, `source_at`, `stale`, `replaced_by`, `valid_until` (1.13.0)
- `scoring.py`, `eval_runner.py`, `cli_memory.py`, and `retriever.py` all still reference old field names: `category`, `valid_at`, `invalid_at`
- These callers either fail silently or return incorrect results against the new graph schema
- This spec is a field name update pass — no logic changes, no new features

## TODO

### 1. `scoring.py` — `match_facts`

Replace the match key:

- Old: `category`, `from_entity`, `to_entity`
- New: `edge_label`, `fact_type`, `from_entity`, `to_entity`

A fact is a match when `edge_label`, `fact_type`, `from_entity`, and `to_entity` all agree (case-insensitive). Exact string match on fact text is not required for a match — entity references and type/label are the identity.

`valid_until` replaces `valid_at` in extracted and expected facts for temporal scoring.

Confidence mismatch (`inferred` vs `stated`) does not fail a fact match — log as a warning only.

Score output reports precision, recall, F1 per edge label (AFFILIATED / ASSERTED / TRANSITIONED) and overall.
