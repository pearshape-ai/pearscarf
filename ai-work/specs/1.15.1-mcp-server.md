# MCP server bootstrap: FastMCP, named key auth, and system integration

## Context

- `context_query.py` exists with all core query functions (1.15.0)
- No MCP server exists yet ŌĆö external agents have no way to call PearScarf
- The system currently starts via `psc run` (REPL) and `psc discord` (Discord bot) ŌĆö both start indexer, experts, and workers as background threads
- The MCP server follows the same pattern: starts as a background thread in both `psc run` and `psc discord`, and also supports standalone `psc mcp start`
- PearScarf is intended for cloud deployment ŌĆö agents on remote systems (Claude, OpenClaw, etc.) must be able to call it

## Library choice

Use **FastMCP** (`fastmcp` package) on top of the official Anthropic `mcp` SDK. FastMCP is the community-standard thin wrapper that reduces tool registration to a single decorator per tool. No custom protocol implementation.

Transport: **HTTP/SSE** ŌĆö remote accessible, standard for cloud deployment.

### 2. Create `mcp_server.py`

A new module at `pearscarf/mcp_server.py`. Follows the same start/stop interface as `Indexer` ŌĆö it is a background thread.

```python
class MCPServer:
    def start(self) -> None: ...  # starts server in background thread
    def stop(self) -> None: ...   # graceful shutdown
```

**FastMCP setup:**

```python
from fastmcp import FastMCP

mcp = FastMCP("PearScarf")
```

**HTTP/SSE transport:** bind to host `0.0.0.0`, port from config (`MCP_PORT`, default `8090`).

**Auth middleware:** every request must include `Authorization: Bearer <key>`. Validate via `store.validate_mcp_key(raw_key)`. Reject with HTTP 401 if missing or invalid. This middleware runs before any tool handler.

**Health check endpoint:** `GET /health` returns `{"status": "ok", "version": "<version>"}` ŌĆö no auth required. Used by load balancers and deployment health checks.

**No tools registered in this spec.** Tools are added in 1.15.2ŌĆō1.15.5.

Following the same pattern as `psc curator`:


## What NOT to do

- Do not register any tools in this spec ŌĆö tool registration is 1.15.2ŌĆō1.15.5
- Do not implement the MCP server as a separate process ŌĆö it is a background thread, same as the indexer
- Do not store raw API keys ŌĆö store SHA-256 hashes only, return the raw key once on creation
- Do not require auth on the `/health` endpoint ŌĆö health checks must work without credentials
- Do not use stdio transport ŌĆö HTTP/SSE only, the system is designed for remote cloud deployment
- Do not skip wiring into `psc run` and `discord_bot.py` ŌĆö the MCP server must start automatically when the system starts
