# Graph-based eval — replaces flat-file eval

## Context

- Current `psc eval` runs LLM extraction against flat files and scores the raw output — no graph involved
- This approach was useful for calibrating the ground truth and debugging the extraction prompt — that work is done
- `psc extract-test` covers the remaining "run extraction against a record and inspect output" debugging use case
- The flat-file eval mode has no further purpose and is removed
- `psc eval` becomes the graph-based eval exclusively: ingest seed → ingest records → wait for indexer → query graph → score against ground truth
- `parse_record_file` tool from the ingest expert is fully implemented and tested by v1.11.3 — `eval_runner.py` calls its `execute()` method directly, no agent or subprocess needed
- Dataset folder structure is defined in pearscarf-eval and must be respected exactly

## TODO

### `eval_runner.py` — replace with graph eval pipeline

Remove `run_eval()` entirely. Replace with `run_graph_eval(dataset_path, verbose)`. Steps in order:

**Step 1 — Seed**
- Check for `seed.md` in the dataset folder
- If present, call `store.save_ingest()` with the seed file content
- If absent, skip with a warning — eval proceeds without a seed

**Step 2 — Ingest records**
- Instantiate `ParseRecordFileTool` directly from `pearscarf.experts.ingest`
- Call `execute()` once per record type subfolder: `emails/` → `email`, `issues/` → `issue`, `issue_changes/` → `issue_change`
- Abort if any call returns a schema validation error — all-or-nothing per the tool's existing behavior

**Step 3 — Wait for indexer**
- Poll `records WHERE indexed = FALSE AND classification = 'relevant'` until count reaches zero
- Poll interval: 2 seconds
- Timeout: configurable, default 120 seconds
- Print progress: "Waiting for indexer... N records remaining"
- Abort with error if timeout reached — do not score a partial graph

**Step 4 — Query graph and score**
- For each record in `ground_truth.json`, query the graph for entities and facts sourced from that record via `graph.get_nodes_by_source_record(record_id)`
- Build extracted entities and facts from graph results — same shape as current scoring input
- Score using existing `scoring.score_record()` — no changes to scoring logic
- ERA and temporal accuracy scored using `resolution_pairs` and `temporal_assertions` from ground truth, resolved against the graph

**Step 5 — Report and write results**
- Use existing `eval_report.print_report()` and `eval_report.write_results()`
- Results written to `dataset_path/results/` with filename `{pearscarf_version}_{dataset_version}.json`
