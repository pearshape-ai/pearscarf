# Entity resolution ‚Äî IDENTIFIED_AS edges

## Context

- 1.12.3 makes resolution decisions but does not record them on the graph
- Without persisting resolution decisions, the same surface form triggers the LLM judge on every record ‚Äî expensive and inconsistent
- `IDENTIFIED_AS` edges record confirmed resolutions as first-class graph state so future retrievals find them without an LLM call
- This spec wires edge writes after every confirmed resolution decision ‚Äî LLM match or HIL confirmation

## TODO

### `graph.py` ‚Äî `create_identified_as_edge()`

- New function that writes an `IDENTIFIED_AS` edge on a resolved entity node
- Edge properties: `surface_form` (the raw name string that was resolved, e.g. "David", "Jim", "J. Whitfield"), `source_record` (record ID), `source_type` (record type), `confidence` (`stated` for exact match, `inferred` for LLM match, `confirmed` for HIL), `resolved_at` (timestamp), `reasoning` (the LLM or human reasoning string)
- Edge direction: from the resolved entity node to itself ‚Äî a self-edge ‚Äî this avoids creating a separate alias node and keeps alias lookup simple
- `IDENTIFIED_AS` is already in `FACT_CATEGORIES` in `graph.py` ‚Äî no new category needed

### `indexer.py` ‚Äî write `IDENTIFIED_AS` after match decisions

- After `_resolve_entity_with_llm()` returns `match`, call `graph.create_identified_as_edge()` on the matched entity with `confidence: inferred`
- After exact name + email/domain match fast path, call `graph.create_identified_as_edge()` with `confidence: stated` ‚Äî only if the surface form differs from the canonical name (e.g. "Jim" vs "James Whitfield") ‚Äî skip if surface form equals canonical name
- After HIL confirmation (1.12.5 will trigger re-index, but the edge write belongs here) ‚Äî write with `confidence: confirmed`
- Do not write `IDENTIFIED_AS` for `new` decisions ‚Äî no alias to record
