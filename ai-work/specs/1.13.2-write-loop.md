# Write loop: literal dup check before write

## Context

- `graph.py` now has `find_existing_fact_edge` and `mark_fact_stale` (1.13.0) — these are used by the verification agent, not the write loop
- `indexer._process_record` now reads `edge_label`, `fact_type`, `source_at` from extracted facts (1.13.1)
- But `_process_record` still calls `create_fact_edge` with no duplicate check — re-indexing the same record produces duplicate edges
- All semantic dedup, supersession, and staleness logic belongs to the verification & augmentation agent, not the write loop
- The graph is eventually consistent: structurally correct at write time, semantically correct after the verification agent runs

## Separation of concerns

**Write loop (this spec):** structural correctness only. Never write the same literal fact twice from the same record. Always write otherwise. Fast, deterministic, no LLM calls.

**Verification & Augmentation Agent:** semantic correctness. Supersession, staleness, deduplication, enrichment, conflict resolution. Async, periodic, non-blocking.

Between writes and the verification agent running, the graph may contain redundant or semantically equivalent facts. LLMs consuming this context treat them as weak signal — they are just noise, not corruption.

## TODO

### 1. Add literal dup check before `create_fact_edge`

Before calling `create_fact_edge`, query for an existing edge matching all of:
`(from_id, edge_label, fact_type, to_id, source_record, fact, role)`

- `fact` text is part of the key — two distinct facts from the same record to the same target must never be conflated (e.g. two separate commitments to the same entity in the same email)
- `role` is part of the key for AFFILIATED facts — Mark as Head of Operations at Meridian and Mark as Head of Partnerships at Meridian are two different edges, not duplicates

**Match found:**
- Append `source_record` to `r.source_records` if not already present
- Skip `create_fact_edge`
- Continue to next fact
- Log: `"dup merged: {source_record} already in edge {rid}"`

**No match:**
- Call `create_fact_edge`
- Log: `"wrote new edge: ({edge_label}, {fact_type})"`

This query can be added as a helper on `graph.py` (`find_exact_dup_edge`) or inlined in the indexer — implementer's choice.
