# Context

- The extraction prompt is being iterated on. When the prompt changes, all previously extracted data in Neo4j is stale.
- There's no way to wipe the graph and re-run extraction over existing emails without manually clearing Neo4j and updating Postgres flags.
- `extract-test` exists for dry-run evaluation but doesn't write to the graph.
- This should be a standalone script, not a CLI command — it's a blunt instrument and shouldn't be easy to run accidentally. Better tools (selective re-extraction, diffing, etc.) will come later.

## Add `scripts/reindex.py`

A standalone script that:

1. Wipes all nodes and relationships in Neo4j (full graph clear)
2. Resets `records.indexed = FALSE` for all records where `classification = 'relevant'`
3. Prints what it did (number of nodes deleted, number of records reset)

The Indexer picks up the reset records automatically on its next poll cycle — the script does not run extraction itself.

Requires confirmation before executing (interactive `y/n` prompt). No `--force` flag.

## What NOT to do

- No CLI command — this is a script only (`python scripts/reindex.py`)
- No selective re-indexing (by record, by date, by entity type) — that's a future version
- No Qdrant clearing — vector search isn't wired yet
- No changes to the Indexer, graph.py, or any application code
