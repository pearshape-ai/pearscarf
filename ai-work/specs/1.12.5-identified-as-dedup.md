# IDENTIFIED_AS edge deduplication via MERGE

## Context

- Every time a record is indexed, the resolution loop creates a new `IDENTIFIED_AS` edge for each resolved surface form
- This produces duplicate self-edges for the same `(entity, surface_form)` pair across multiple source records
- `create_identified_as_edge()` in `graph.py` currently uses `CREATE`, not `MERGE`
- The edge should be a unique alias record, not a per-record event log

## TODO

- Change `create_identified_as_edge()` to `MERGE` on `(entity_id, surface_form)` — one edge per unique alias
- On first creation, set all properties: `fact`, `confidence`, `reasoning`, `source_record`, `source_type`, `category`, `created_at`, `resolved_at`
- On subsequent matches, update `resolved_at` to the latest resolution timestamp and append the new `source_record` to a `source_records` list property on the edge
- `source_record` (singular) stays on the edge as the first-seen record for backwards compatibility; `source_records` accumulates all

## What NOT to do

- Do not change the edge schema visible to the retriever — `source_record`, `confidence`, `category` stay as-is
- Do not touch the resolution logic in `indexer.py` — this change is entirely in `graph.py`
- Do not deduplicate across different surface forms — `"Priya"` and `"P. Nair"` are distinct aliases and each gets its own edge
