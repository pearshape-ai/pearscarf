## Update extraction prompt (`pearscaff/prompts/extraction.md`)

Replace the three-array output (entities, relationships, facts) with two arrays: `entities` and `facts`.

Every fact has:
- `category` — one of the defined categories
- `fact` — the human-readable fact text
- `from_entity` — the source entity name
- `to_entity` — the target entity name, or null if single-entity fact
- `confidence` — stated or inferred
- `valid_at` — ISO date string if a specific date is referenced, otherwise null

The prompt should include the available categories with one-line descriptions so the LLM picks the right one:

- WORKS_AT — person employed at or affiliated with company
- FOUNDED — person founded company
- MANAGES — person leads or manages project
- PART_OF — project belongs to company, or subsidiary relationship
- MEMBER_OF — person is part of project or event
- COMMUNICATED — exchange between people or between person and company
- MENTIONED_IN — entity referenced in a record (fallback when nothing else fits)
- STATUS_CHANGED — status update, stage transition
- COMMITTED_TO — deadline, deliverable, promise, agreement
- DECIDED — decision made
- BLOCKED_BY — dependency, blocker
- EVALUATED — considering, reviewing, comparing

Old "relationships" become facts: "Michael works at Acme" → category WORKS_AT, from_entity "Michael Chen", to_entity "Acme".

Old "facts" become facts: "contract renews March 2026" → category COMMITTED_TO, from_entity "Acme", to_entity null, valid_at "2026-03-01".

New JSON output shape:

```
{
  "entities": [
    {"type": "person", "name": "Michael Chen", "metadata": {"email": "...", "role": "..."}}
  ],
  "facts": [
    {
      "category": "WORKS_AT",
      "fact": "works at Acme as VP Engineering",
      "from_entity": "Michael Chen",
      "to_entity": "Acme",
      "confidence": "stated",
      "valid_at": null
    },
    {
      "category": "COMMITTED_TO",
      "fact": "contract renewal deadline is March 2026",
      "from_entity": "Acme",
      "to_entity": null,
      "confidence": "stated",
      "valid_at": "2026-03-01"
    }
  ]
}
```

Keep all existing prompt guidance: entity types, normalization rules, automated email handling, "extract when / do not extract when" per entity type, confidence levels. Only the output format changes.

## Update extraction test script (`extract_test.py`)

The test script sends emails through the prompt and prints results. Ensure it:

- Parses the new two-array JSON shape (entities + facts, no relationships)
- Pretty-prints facts grouped by category for easy scanning
- Flags any fact where `from_entity` doesn't match a name in the entities array (entity name mismatch — the most common extraction bug)
- Flags any fact with an unrecognized category (typo or hallucinated category)
- Still works with the same invocation pattern (pull email from Postgres or pass inline)
