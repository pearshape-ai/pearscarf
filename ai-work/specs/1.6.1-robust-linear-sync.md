- Capture project and label details — resolve Linear project IDs and label IDs to human-readable names and store them

## Issue record completeness

Make sure the stored issue record captures everything the extraction prompt will need:
- Title and description (the main content)
- Comments (threaded, with author names)
- Status, priority, assignee name
- Project name, label names
- Linear URL for reference
- Created and updated timestamps

The `_build_content` pattern from the Indexer (which assembles email text for extraction) should have an equivalent for issues — assembling title + description + comments into a single text block ready for the extraction prompt.

## Triage integration

Issues flowing through polling should trigger the same triage flow as emails — the worker presents them to the human and asks "is this relevant?" The classification goes on the record, and only relevant issues will be extracted in the next iteration.

For the initial bulk load, consider a batch mode where the human can classify multiple issues at once instead of one-by-one triage through Discord.
