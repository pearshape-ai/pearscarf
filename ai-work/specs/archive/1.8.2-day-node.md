# Context

- Graph model is being rebuilt. Facts will become edges, Day nodes are one of the two node types (alongside entity nodes).
- Day nodes represent calendar days in the user's local timezone. They serve as the structural endpoint for single-entity facts — facts that involve only one entity need a second node to form an edge, and the Day node is that endpoint.
- Day nodes are created lazily — the first fact that falls on a given day triggers creation.
- All timestamps stored in UTC. Day node identity derived by converting UTC to local timezone.
