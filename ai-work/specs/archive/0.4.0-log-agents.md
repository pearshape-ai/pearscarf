## What to Log

Every log entry should include:
- Timestamp
- Agent name (e.g. `worker`, `gmail_expert`)
- Session ID (if applicable, `--` if not)
- Entry type: `action`, `message_sent`, `message_received`, `reasoning`, `thinking`, `tool`, `result`, `error`

Log the full detail of every model interaction:
- `[reasoning]` — the agent's high-level decision about what to do next
- `[thinking]` — the model's internal thought process after observing a result
- `[tool]` — the exact tool call with arguments
- `[result]` — the tool's return value
- `[message_sent]` / `[message_received]` — inter-agent messages
- `[error]` — anything that went wrong

## Format

Verbose, human-readable, one entry per line. The log should read like a full transcript of the agent's brain — every decision, every call, every observation.

```
[2026-02-27T10:30:01Z] [gmail_expert] [ses_001] [action] polling for unread emails
[2026-02-27T10:30:01Z] [gmail_expert] [ses_001] [tool] gmail_get_unread({ "limit": 5 })
[2026-02-27T10:30:02Z] [gmail_expert] [ses_001] [result] gmail_get_unread: found 3 unread emails
[2026-02-27T10:30:02Z] [gmail_expert] [ses_001] [thinking] 3 unread emails found. First email is from investor@acme.com with subject "Q3 Follow-up" — flagging as urgent based on VIP sender list.
[2026-02-27T10:30:02Z] [gmail_expert] [ses_001] [message_sent] to=worker, action=new_email, reasoning="New email from investor@acme.com, flagging as urgent"
[2026-02-27T10:30:03Z] [worker] [ses_001] [message_received] from=gmail_expert, action=new_email
[2026-02-27T10:30:03Z] [worker] [ses_001] [reasoning] Email marked urgent by expert. Human should be notified immediately.
[2026-02-27T10:30:03Z] [worker] [ses_001] [message_sent] to=human, action=notify
[2026-02-27T10:30:10Z] [worker] [ses_001] [message_received] from=human, action=mark_as_read
[2026-02-27T10:30:10Z] [worker] [ses_001] [reasoning] Human wants this email marked as read. Forwarding to gmail_expert.
[2026-02-27T10:30:10Z] [worker] [ses_001] [message_sent] to=gmail_expert, action=mark_as_read, data={ "email_ids": ["msg_42"] }
[2026-02-27T10:30:11Z] [gmail_expert] [ses_001] [message_received] from=worker, action=mark_as_read
[2026-02-27T10:30:11Z] [gmail_expert] [ses_001] [tool] gmail_mark_as_read({ "email_ids": ["msg_42"] })
[2026-02-27T10:30:12Z] [gmail_expert] [ses_001] [result] gmail_mark_as_read: Sent mark-as-read keyboard shortcut (Shift+I).
[2026-02-27T10:30:12Z] [gmail_expert] [ses_001] [thinking] Done! The email has been marked as read. This was a business development email from Michael offering additional development resources for the upcoming project cycle.
[2026-02-27T10:30:12Z] [gmail_expert] [ses_001] [message_sent] to=worker, action=mark_as_read_complete, reasoning="Email msg_42 marked as read successfully"
[2026-02-27T10:30:15Z] [worker] [--] [action] polling for unread messages, none found
```

## Requirements

- Append-only — never overwrite, never truncate.
- All agents write to the same file so the timeline is unified.
- Entries without a session context use `[--]` in place of session ID.
- Log every tool call, every tool result, every thinking step, every reasoning step, every message sent and received, every error.
- The goal is full reproducibility — reading the log should tell you exactly what every agent did and why.
