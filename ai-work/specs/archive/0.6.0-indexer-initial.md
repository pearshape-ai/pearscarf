## Database Schema

Add to existing SQLite database:

```sql
-- Flag on existing records table
ALTER TABLE records ADD COLUMN indexed INTEGER NOT NULL DEFAULT 0;

-- Entity type registry (manually populated, read by Indexer at runtime)
entity_types (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,           -- "person", "company"
    description TEXT,             -- guidance for LLM on how to identify this type
    extract_fields TEXT,          -- JSON: what fields to extract for this type
    added_at TEXT NOT NULL
)

-- Graph nodes
entities (
    id TEXT PRIMARY KEY,
    type TEXT NOT NULL,           -- references entity_types.name
    name TEXT NOT NULL,
    metadata TEXT,                -- JSON, flexible extra info (email, domain, role, etc.)
    created_at TEXT NOT NULL
)

-- Graph edges
edges (
    id TEXT PRIMARY KEY,
    from_entity TEXT NOT NULL REFERENCES entities(id),
    to_entity TEXT NOT NULL REFERENCES entities(id),
    relationship TEXT NOT NULL,   -- "sent", "works_at", "mentioned_in", etc.
    source_record TEXT REFERENCES records(id),
    created_at TEXT NOT NULL
)

-- Facts table (living state, Mem0-style)
facts (
    id TEXT PRIMARY KEY,
    entity_id TEXT NOT NULL REFERENCES entities(id),
    attribute TEXT NOT NULL,
    value TEXT NOT NULL,
    source_record TEXT REFERENCES records(id),
    updated_at TEXT NOT NULL
)
```

## Indexer Agent

### What it is

An expert agent that runs as a background process. It does not participate in sessions or communicate with other agents via messages. It reads from the records table and writes to the graph tables.

### How it runs

```
Indexer (background loop):
    ↓
    Poll: SELECT * FROM records WHERE indexed = 0
    ↓
    For each unprocessed record:
        1. Read full content from typed table (e.g. emails)
        2. Build extraction prompt from entity_types registry
        3. Send to LLM, get structured JSON response
        4. Deduplicate entities against existing graph
        5. Write new/updated entities, edges, facts
        6. Set records.indexed = 1
    ↓
    Sleep. Poll again.
```

### Extraction prompt

Built at runtime from entity_types table. Example assembled prompt:

```
Given this record, extract all entities, relationships, and facts.
Respond in JSON only, no other text.

Entity types to extract:
- person: A human being. Look for names, email addresses, roles, titles. Fields: name, email, role
- company: A business or organization. Look for company names, domains, industries. Fields: name, domain

Record:
From: Michael Chen <michael@acmecorp.com>
Subject: Q3 Invoice - $5,000
Body: Hi Gev, attached is the Q3 invoice...

Respond with:
{
  "entities": [{ "type": "...", "name": "...", "metadata": {} }],
  "relationships": [{ "from": "...", "to": "...", "type": "..." }],
  "facts": [{ "entity": "...", "attribute": "...", "value": "..." }]
}
```

### Entity resolution (v1)

Simple matching before creating new entities:

- **Person**: match on exact name + email, or exact name + same company
- **Company**: match on exact name or domain

```sql
-- Check if person exists
SELECT id FROM entities 
WHERE type = 'person' 
AND (name = ? OR metadata LIKE '%' || ? || '%')

-- Check if company exists
SELECT id FROM entities
WHERE type = 'company'
AND (name = ? OR metadata LIKE '%' || ? || '%')
```

If match found, reuse existing entity ID and add new edges. If no match, create new entity.

### Fact deduplication

When writing facts, check if a fact already exists for that entity + attribute:

- If exists: update value, source_record, updated_at
- If not: insert new fact
