# Problems

## 1. Crash on exit

Typing `exit` crashes with a greenlet thread error. The issue is `gmail_manager.close()` tries to save Playwright browser state after the event loop or thread has already exited. The cleanup sequence needs to handle this gracefully — save state before tearing down the async context, or catch the error and skip state saving if the thread is gone.

Stack trace for reference:
```
File "pearscaff/cli.py", line 46, in run
    gmail_manager.close()
File "pearscaff/experts/gmail.py", line 421, in close
    self.save_state()
File "pearscaff/experts/gmail.py", line 417, in save_state
    self._context.storage_state(path=str(STORAGE_STATE_PATH))
greenlet.error: cannot switch to a different thread (which happens to have exited)
```

## 2. Response text spacing is broken

Worker responses display with erratic indentation and line breaks. The text should render flush left with normal line wrapping, matching the same alignment as the `[ses_XXX] worker >` prefix.
