# Context

- Mem0 integration is in place, writing memories and graph data to Neo4j
- No way to inspect what's stored without writing ad-hoc code
- Need a set of tools to see what's in the memory layer — what entities exist, what relationships were extracted, what memories come back for a query
- Should be usable from the REPL and CLI, not a separate dashboard or webapp

# TODO

## Memory Inspection Commands

New CLI commands under `pearscaff memory` (or `ps memory`):

### List memories

```
ps memory list
ps memory list --limit 20
```

- Shows all stored memories for the current user
- Each memory: ID, content, created_at
- Default limit 10, configurable

### Search memories

```
ps memory search "Acme Corp"
ps memory search "fundraising" --limit 5
```

- Runs Mem0 search, shows results with relevance
- Each result: memory content, metadata (record_id, type, source)
- If graph memory enabled, also shows related entities

### Show entity

```
ps memory entity "Michael Chen"
ps memory entity "Acme Corp"
```

- Looks up a specific entity in the graph
- Shows:
  - Entity name, type
  - Connected entities (who/what is linked)
  - Relationships (works_at, sent_email, partner, etc.)
  - Related memories

### Graph overview

```
ps memory graph
ps memory graph --limit 50
```

- High-level summary of what's in the graph
- Total entities, total relationships
- Entity types breakdown (e.g. 12 people, 5 companies, 3 projects)
- Most connected entities (which nodes have the most edges)

### Memory history for a record

```
ps memory record email_71
```

- Shows what memories were extracted from a specific record
- Links back to the raw email in the System of Record
- Shows entities and relationships that came from this record

## REPL Integration

Same commands available as REPL commands:

```
[ses_001] you > /memory search "Acme Corp"
[ses_001] you > /memory entity "Michael Chen"
[ses_001] you > /memory graph
```

- Output formatted for the REPL (compact, readable)
- Does not create a session or involve the worker — direct data access

## Neo4j Browser

- Not part of this spec, but document in getting-started.md that Neo4j Browser is available at localhost:7474 for visual graph exploration
- Useful for ad-hoc Cypher queries beyond what the CLI tools cover
