# graph.py: new fact edge schema + write/read helpers

## Context

- `graph.py` is the sole interface between the indexer and the graph db ‚Äî all reads and writes go through it
- Current fact edges use a flat `category` string (WORKS_AT, COMMITTED_TO, etc.) with `valid_at` / `invalid_at` temporal model
- The new model uses `edge_label` (AFFILIATED / ASSERTED / TRANSITIONED) as the relationship type plus a `fact_type` property, with `source_at` / `recorded_at` bi-temporal fields and `stale` / `replaced_by` for supersession
- This spec rewires `graph.py` only ‚Äî `indexer.py`, prompts, and all readers intentionally break here and are fixed in 1.13.1 and 1.13.3
- `create_identified_as_edge` is excluded ‚Äî it has its own schema and is not a fact edge

### 2. Update `create_fact_edge` signature and written properties

New signature:
```
create_fact_edge(
    from_node_id, to_node_id,
    edge_label, fact_type, fact,
    confidence, source_record, source_type,
    source_at, valid_until=None
)
```

Properties written to the edge:
- `fact` ‚Äî unchanged
- `edge_label` ‚Äî replaces `category` as the Neo4j relationship type (used in `apoc.create.relationship`)
- `fact_type` ‚Äî new property on the edge
- `confidence` ‚Äî unchanged
- `source_record` ‚Äî unchanged
- `source_records` ‚Äî list, initialized to `[source_record]`
- `source_type` ‚Äî unchanged
- `source_at` ‚Äî event time, passed in (replaces `valid_at`)
- `recorded_at` ‚Äî transaction time, set to `_now()` on write
- `stale` ‚Äî boolean, initialized to `false`
- `replaced_by` ‚Äî initialized to `null`
- `valid_until` ‚Äî deadline/expiry edge property, `null` by default
- `created_at` ‚Äî keep for audit, still set to `_now()`
- Drop `invalid_at` entirely

### 3. Add `find_existing_fact_edge`

```
find_existing_fact_edge(from_id, edge_label, fact_type, to_id)
‚Üí dict {id, fact, source_at, stale} | None
```

- Query: match outgoing edge of type `edge_label` from `from_id` to `to_id` where `r.fact_type = fact_type` and `r.stale = false` and `r.replaced_by IS NULL`
- Returns the first match or `None`
- Used by the write loop in 1.13.2
