# Convenience tools: get_entity_context, get_current_state

## Context

- All five primitive tools are live (1.15.2, 1.15.3)
- Convenience tools are pre-composed for common agent tasks — no new graph queries, just preset parameters over primitives
- `get_entity_context` composes `get_facts` + `get_connections` into one structured package
- `get_current_state` is `get_facts` scoped to AFFILIATED — a named concept that helps agents self-select the right tool

### 1. Register `pearscarf.get_entity_context`

```python
@mcp.tool(description=(
    "Get a full picture of an entity — all current facts and direct connections. "
    "Use this before acting on behalf of or about a person, company, or project. "
    "Returns facts in chronological order by default, or grouped by edge label with format='clustered'."
))
def get_entity_context(
    entity_name: str,
    format: str = "chronological",
    include_stale: bool = False,
) -> dict:
```

Parameters:
- `entity_name` — surface form (required). Resolved internally.
- `format` — `"chronological"` (default) or `"clustered"`
- `include_stale` — include superseded facts. Default `false`.

Resolves `entity_name`. If not found, returns `{"error": "not_found", "name": entity_name}`.

Calls:
- `context_query.get_facts(entity_id, include_stale=include_stale)`
- `context_query.get_connections(entity_id, max_depth=1, include_stale=include_stale)`

**Chronological format** (`format="chronological"`):
```json
{
  "entity": {"id": "...", "name": "Marcus Webb", "type": "person", "metadata": {...}},
  "facts": [<canonical fact objects, ordered by source_at ascending>],
  "connections": [{"name": "Acme Corp", "type": "company"}, ...],
  "count": 7
}
```

**Clustered format** (`format="clustered"`):
```json
{
  "entity": {"id": "...", "name": "Marcus Webb", "type": "person", "metadata": {...}},
  "facts": {
    "AFFILIATED": [<fact objects>],
    "ASSERTED": [<fact objects>],
    "TRANSITIONED": [<fact objects>]
  },
  "connections": [{"name": "Acme Corp", "type": "company"}, ...],
  "count": 7
}
```

Both formats include the same canonical fact objects — only the arrangement differs. `count` is the total number of facts regardless of format.

If `format` is not `"chronological"` or `"clustered"`, return:
```json
{"error": "invalid_format", "valid_values": ["chronological", "clustered"]}
```

### 2. Register `pearscarf.get_current_state`

```python
@mcp.tool(description=(
    "Get what is structurally true about an entity right now — "
    "current role, employer, project memberships. "
    "Returns AFFILIATED facts only. "
    "Use when you need to know who someone works for or what projects they belong to."
))
def get_current_state(entity_name: str) -> dict:
```

Parameters:
- `entity_name` — surface form (required). Resolved internally.

Resolves `entity_name`. If not found, returns `{"error": "not_found", "name": entity_name}`.

Calls `context_query.get_facts(entity_id, edge_label="AFFILIATED", include_stale=False)`.

Returns flat structure — no format parameter:
```json
{
  "entity": {"id": "...", "name": "Marcus Webb", "type": "person"},
  "affiliations": [<canonical fact objects, AFFILIATED only, stale=false>],
  "count": 2
}
```

Key distinction from `get_entity_context`: `get_current_state` is point-in-time structure only. Use `get_entity_context` when you need the full picture including claims and transitions.

## What NOT to do

- Do not add new graph queries — both tools compose existing `context_query.py` functions
- Do not add a `format` parameter to `get_current_state` — flat structure, no format needed
- Do not include Day nodes in `connections` — same rule as `get_connections`
- Do not implement `get_open_commitments`, `get_open_blockers`, or `get_recent_activity` here — those are 1.15.5
