# Eval runner end-to-end

---

## Change 1 — `graph.py`: expose `valid_until` from fact edges

**Why:** scorer needs `valid_until` to evaluate temporal assertions; it's on the edge in Neo4j but never returned

**File:** `graph.py` — `get_nodes_by_source_record` Cypher RETURN clause

```
# Before
RETURN r.fact AS fact, r.edge_label AS edge_label, ...

# After
RETURN r.fact AS fact, r.edge_label AS edge_label, ..., r.valid_until AS valid_until
```

Also add `"valid_until": row["valid_until"]` to the returned item dict.

---

## Change 2 — `_build_extracted_from_graph`: add `stale` and `valid_until` to fact dict

**Why:** both fields are now returned by `get_nodes_by_source_record` but dropped before reaching the scorer

**File:** `eval_runner.py` — `_build_extracted_from_graph`

```python
# Before
fact = {
    ...
    "source_at": item.get("source_at", ""),
}

# After
fact = {
    ...
    "source_at": item.get("source_at", ""),
    "stale": item.get("stale", False),
    "valid_until": item.get("valid_until"),
}
```

## Change 3 — `_print_verbose_graph`: fix stale field names

**Why:** verbose mode shows `?` for every fact label and no valid date — `category` and `valid_at` no longer exist

**File:** `eval_runner.py` — `_print_verbose_graph`, expected facts block

```python
# Before
valid = f"  (valid_at: {f['valid_at']})" if f.get("valid_at") else ""
print(f"  [{f.get('confidence', '?')}] {f.get('category', '?')}: ...")

# After
valid = f"  (valid_until: {f['valid_until']})" if f.get("valid_until") else ""
label = f"{f.get('edge_label', '?')}/{f.get('fact_type', '?')}"
print(f"  [{f.get('confidence', '?')}] {label}: ...")
```

**File:** `eval_runner.py` — `_print_verbose_graph`, graph facts block

```python
# Before
valid = f"  (source_at: {f['source_at']})" if f.get("source_at") else ""

# After — prefer valid_until, fall back to source_at
valid = f"  (valid_until: {f['valid_until']})" if f.get("valid_until") else (
    f"  (source_at: {f['source_at']})" if f.get("source_at") else ""
)
```

## Change 5 — per-label F1 aggregation

**Why:** `score_record` now returns per-label counts but the runner never aggregates or reports them

**File:** `eval_runner.py` — after the existing `aggregate` block

```python
# Add after aggregate["noise_rejection_rate"] = ...
per_label = {}
for label in ("affiliated", "asserted", "transitioned"):
    lm = sum(r.get(f"{label}_matched", 0) for r in per_record.values())
    le = sum(r.get(f"{label}_extracted", 0) for r in per_record.values())
    lx = sum(r.get(f"{label}_expected", 0) for r in per_record.values())
    lp = scoring.precision(lm, le)
    lr = scoring.recall(lm, lx)
    per_label[label] = {"precision": lp, "recall": lr, "f1": scoring.f1(lp, lr)}
aggregate["per_label_f1"] = per_label
```
