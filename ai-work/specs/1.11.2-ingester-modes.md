# Ingest expert — modes, prompt improvements, and pipeline wiring

## Context

- v1.11.1 fully implemented ingest expert tools and the indexer ingest branch
- `psc expert ingest` exists as a standalone interactive agent but has no invocation modes
- The seed pipeline is end-to-end but untested — no way to invoke it from the CLI yet
- `prompts/ingest.md` is functional but does not specify what a useful reply looks like, and does not tell the agent how to detect which mode it is in from the pre-built message
- This version wires up the ingest expert modes and improves the prompt so the full seed pipeline can be exercised from the terminal
- Goal: `psc expert ingest --seed <file>` runs the seed through the pipeline and the graph gets populated

## TODO

### Prompt — `prompts/ingest.md`

Two changes only:

- **Mode detection**: tell the agent to recognize the pre-built message format to pick the right tool without reasoning. Seed mode message starts with `"Ingest seed file:"`, record mode message starts with `"Ingest <type> records from:"`.
- **Reply content spec**: tell the agent what a useful reply includes:
  - Seed mode: the `record_id` assigned and a summary of what was found in the file (entity count per type, fact count)
  - Record mode: count of records saved, count of duplicates skipped, record IDs assigned

### CLI — ingest expert modes

- `psc expert ingest` gains two optional flags:
  - `--seed <file>` — seed mode: sends pre-built message `"Ingest seed file: <path>"` to the agent, prints result, exits
  - `--record <file> --type <record_type>` — record mode: sends pre-built message `"Ingest <type> records from: <path>"` to the agent, prints result, exits
- Without flags, falls through to interactive mode as today
- Both flags follow the same non-interactive pattern as `psc expert gmail --auth`

### Indexer — verify ingest branch is reachable

- `ingest` records written by `parse_seed` must have `classification = 'relevant'` and `indexed = false` so the indexer picks them up on its next poll cycle
- Confirm `_build_content()` ingest branch reads `raw` from `records` table correctly
- Confirm `_extract()` routes `type == 'ingest'` to `prompts/ingest_extraction.md`

### End-to-end test path

The full sequence that must work after this version:

1. `psc expert ingest --seed datasets/base-scenario/seed.md`
2. Ingest expert detects seed mode from message, calls `parse_seed`
3. Seed file saved as `ingest` record in Postgres, agent replies with `record_id` and entity/fact summary
4. Indexer picks up the record on next poll cycle
5. Indexer loads `ingest_extraction.md`, calls LLM with seed content
6. Entities and facts written to Neo4j graph
7. `psc memory graph` shows expected entity counts
