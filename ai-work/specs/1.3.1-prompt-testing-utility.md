# Context

- The extraction prompt is being rebuilt from scratch. Before wiring anything into the indexer or storage, the prompt needs to be tested in isolation against real emails.
- The fastest iteration loop is: run prompt against an email, read the JSON output, judge quality, tweak prompt, repeat.
- This should work outside the full PearScaff system — no agents, no bus, no sessions. Just email in, structured extraction out, printed to terminal.

## Standalone extraction test script

A script that:
- Takes a record ID as argument (e.g. `email_042`), or runs against all classified-relevant emails if no argument given
- Loads the email content from Postgres (using the existing store/db layer)
- Loads the extraction prompt from `pearscaff/prompts/extraction.md`
- Sends it to Claude and prints the result — the raw JSON extraction (entities, relationships, facts)
- Prints the email content above the extraction so you can compare side by side

When run against multiple emails (no argument), it should process them one at a time and print each result clearly separated, not dump everything at once.

If LangSmith tracing is enabled in the environment, extraction calls should be traced. Use the existing tracing utilities. Not a priority — just wire it if it's easy.

## What this is for

Prompt iteration. You edit `pearscaff/prompts/extraction.md`, run the script, see if the output improved. No storage writes, no graph updates, nothing persisted. Pure evaluation.

## What NOT to do

- No writing to graph tables, vector store, or Neo4j
- No changes to the indexer
- No eval scoring or automated comparison — that comes later
- No new dependencies
