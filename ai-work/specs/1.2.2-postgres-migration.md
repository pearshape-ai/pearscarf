# Context

- All application data currently lives in SQLite: sessions, messages, records, emails, entities, edges, facts, entity_types, discord_threads
- Neo4j and Qdrant already run as Docker containers with mounted data volumes
- SQLite was the right choice when everything was single-process and zero-infrastructure
- Now that we have Docker containers for other storage, SQLite is the odd one out
- Postgres is more capable (concurrent writes, better indexing, richer queries) and aligns with the containerized infrastructure pattern
- Graphiti also supports Postgres-backed setups, so this prepares the ground
