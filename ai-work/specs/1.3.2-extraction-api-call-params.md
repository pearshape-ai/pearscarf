## Configure the extraction API call for structured extraction

The LLM call for entity extraction should use settings optimized for consistent, structured JSON output:

- **Temperature 0** — extraction should be deterministic. Same email in, same entities out. No creative variation.
- **System/user prompt split** — the extraction instructions (entity types, output format, what to ignore) go in the system prompt. The email content goes in the user message. This gives the model clearer role separation than stuffing everything into one message.
    - leave the system prompt in the file as is (only remove the section related to user prompt)
    - move the user ptompt to the code as its a fixed text and is not affected by model iteration.
- **Model** — use the same model as the rest of the system (configurable via MODEL env var), but the extraction call should be easy to point at a different model later if needed (e.g. a cheaper model for bulk processing).
- **Max tokens** — cap appropriately for JSON extraction output. The current 2048 is reasonable but should be a named constant, not buried in the call.
