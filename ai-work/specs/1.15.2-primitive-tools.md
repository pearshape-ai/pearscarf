# Primitive tools: find_entity, get_facts, get_connections

## Context

- `context_query.py` exists with `find_entity`, `get_facts`, `get_connections` (1.15.0)
- `mcp_server.py` has a running FastMCP instance with auth and no tools registered (1.15.1)
- This spec registers the first three primitive tools ‚Äî the foundational layer everything else composes from
- The pattern established here (decorator, entity resolution, error shape) is followed by all subsequent tool specs

## Registration pattern

All tools live directly in `mcp_server.py`. Each tool is a FastMCP-decorated function that calls `context_query.py` and handles the entity-not-found case. No business logic in the handler ‚Äî only resolution, dispatch, and error wrapping.

```python
@mcp.tool()
def find_entity(name: str, entity_type: str = None) -> dict:
    results = context_query.find_entity(name, entity_type)
    if not results:
        return {"error": "not_found", "name": name}
    return {"entities": results}
```

**Entity resolution pattern** used by all entity-scoped tools:

```python
matches = context_query.find_entity(entity_name)
if not matches:
    return {"error": "not_found", "name": entity_name}
entity_id = matches[0]["id"]  # use the top match
```

If multiple matches are returned, use the first (highest confidence). Agents that need disambiguation should call `find_entity` first to inspect candidates.

**Error shape** is consistent across all tools:
```json
{"error": "not_found", "name": "<the name passed in>"}
```

### 2. Register `pearscarf.get_facts`

```python
@mcp.tool()
def get_facts(
    entity_name: str,
    edge_label: str = None,
    fact_type: str = None,
    include_stale: bool = False,
    since: str = None,
) -> dict:
```

Parameters:
- `entity_name` ‚Äî surface form (required). Resolved internally.
- `edge_label` ‚Äî optional: `"AFFILIATED"`, `"ASSERTED"`, or `"TRANSITIONED"`
- `fact_type` ‚Äî optional: any valid fact_type within the given edge_label (e.g. `"commitment"`, `"employee"`, `"status_change"`)
- `include_stale` ‚Äî include superseded facts. Default `false`.
- `since` ‚Äî ISO datetime string. Only return facts where `source_at >= since`.

Resolves `entity_name` via `find_entity`. If not found, returns `{"error": "not_found", "name": entity_name}`.

Calls `context_query.get_facts(entity_id, edge_label, fact_type, include_stale, since)`.

Returns:
```json
{
  "entity": {"id": "...", "name": "Marcus Webb", "type": "person"},
  "facts": [<canonical fact objects>],
  "count": 4
}
```

Returns `{"entity": {...}, "facts": [], "count": 0}` if entity resolves but no matching facts exist.

Use when: an agent needs facts about an entity, optionally filtered by type. This is the workhorse ‚Äî `get_current_state`, `get_open_commitments`, and `get_open_blockers` are all specializations of this tool.

### 3. Register `pearscarf.get_connections`

```python
@mcp.tool()
def get_connections(
    entity_name: str,
    edge_label: str = None,
    include_stale: bool = False,
) -> dict:
```

Parameters:
- `entity_name` ‚Äî surface form (required). Resolved internally.
- `edge_label` ‚Äî optional: filter traversal to one edge label
- `include_stale` ‚Äî include stale edges. Default `false`.

Resolves `entity_name`. If not found, returns `{"error": "not_found", "name": entity_name}`.

Calls `context_query.get_connections(entity_id, max_depth=1, include_stale=include_stale, edge_labels=[edge_label] if edge_label else None)`.

Returns:
```json
{
  "entity": {"id": "...", "name": "Marcus Webb", "type": "person"},
  "connections": [
    {"id": "...", "name": "Acme Corp", "type": "company"},
    {"id": "...", "name": "Meridian Deal", "type": "project"}
  ],
  "edges": [<fact edge objects>],
  "count": 2
}
```

Day nodes are excluded from `connections` ‚Äî they are internal anchoring nodes, not meaningful entities to surface to agents.

Use when: an agent needs to understand what entities a person, company, or project is directly connected to.

### 4. Tool descriptions

FastMCP exposes tool descriptions to agents. Each tool needs a clear `description` parameter on the decorator:

```python
@mcp.tool(description=(
    "Resolve a name to a canonical entity in the PearScarf graph. "
    "Use this first when you have a name and need to confirm PearScarf knows this entity, "
    "or when a name could match multiple entities."
))
def find_entity(...):
```

Write descriptions that tell an agent *when* to use the tool, not just *what* it does. Agents decide which tool to call based on descriptions.

## What NOT to do

- Do not add business logic to the MCP handlers ‚Äî handlers call `context_query` and return, nothing else
- Do not accept lists for `edge_label` or `fact_type` ‚Äî single string only, one filter per call
- Do not return raw Neo4j node or edge objects ‚Äî always return the canonical shapes defined in the architecture doc
- Do not expose Day nodes in `get_connections` results ‚Äî internal anchoring only
- Do not implement `get_relationship` or `get_conflicts` here ‚Äî those are 1.15.3
- Do not implement `get_entity_context` here ‚Äî that is 1.15.4, it is a convenience composition of these primitives
