# Context

- `psc extract-test` exists and works: calls the extraction LLM against one or more records, prints structured output to terminal. Used for manual prompt iteration.
- No way to run extraction against a full dataset and get scored results. Eval metrics are defined, ground truth exists, but nothing connects them.
- `psc eval` is `extract-test` at dataset scale — same extraction path, with scoring against ground truth on top.
- Scoring logic and the eval command live in pearscarf alongside the extraction logic they depend on.

## `psc eval` command

```
psc eval --dataset <path>
psc eval --dataset <path> --verbose    # print record content + expected/extracted per record
```

- `--verbose` / `-v` prints per-record debug output: record content, expected entities/facts, extracted entities/facts
- Loads all records from `<path>/data/` subdirectories in chronological order
- Loads ground truth from `<path>/ground_truth.json`
- For each record: runs the same extraction LLM call as `extract-test`
- Scores output against ground truth: entity precision/recall, fact precision/recall, noise rejection
- Aggregates: Extraction Recall, Extraction Precision, Graph Fidelity (F1), Noise Rejection Rate, Entity Resolution Accuracy, Temporal Accuracy
- Prints terminal report
- Writes results to `<path>/results/<pearscarf_version>_<dataset_version>.json`

Dataset version is read from `<path>/dimensions.md` or a `version` field in ground truth. PearScarf version comes from `pearscarf.__version__`.

When `LANGSMITH_TRACING=true` is set in the environment, each extraction call is traced as a LangSmith run under a dedicated eval project. The project name is `pearscarf-eval-<dataset_version>` by default, overridable via `LANGSMITH_PROJECT`. This uses the existing tracing instrumentation — no new tracing code needed. Each record's extraction call appears as a separate trace, tagged with `record_id`, `dataset_version`, and `pearscarf_version` so runs can be compared across versions in the LangSmith dashboard.

ERA and Temporal Accuracy are scored in this version using `resolution_pairs` and `temporal_assertions` from ground truth (populated in pearscarf-eval v0.3.0).

## Scoring module

Scoring functions for entity matching, fact matching, F1, NRR, ERA, and Temporal Accuracy. Separate from `extract-test` and `psc eval` so both can reuse them.

- Entity matching: exact and fuzzy name comparison
- Fact matching: category + from/to entity names
- ERA: evaluated over `resolution_pairs` from ground truth — checks whether surface form variants across records resolve to the same canonical entity name in extraction output, or correctly stay separate for split cases
- Temporal Accuracy: evaluated over `temporal_assertions` from ground truth — checks whether extracted facts carry the correct `valid_at` values matching real-world event times, not ingestion time

## Reporter

Terminal report formatter and JSON results writer. Used by `psc eval`. Same style as `extract-test` terminal output, extended with aggregate scores.

# What NOT to build

- No changes to `extract-test` — it stays as the single-record prompt iteration tool
- No ingestion, no Postgres, no Indexer involvement — dataset records are input files only
- No full-pipeline eval — extraction-only for now
- No ERA or Temporal Accuracy scoring — ground truth annotation is a separate pass
- No multi-dataset comparison — single run only
- No CI integration — manual runs only
