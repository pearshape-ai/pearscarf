# Context

- Full focus shifting to entity extraction quality. Need a clean slate to build extraction properly.
- The current indexer, retriever, graph module, and vector store were built incrementally from v0.6.0 through v1.2.2. The extraction quality was never good enough — that's what the next phase fixes.
- Before building the new extraction pipeline, gut the old storage-writing logic so there's no confusion about what's active and what's dead code.
- Keep all infrastructure config, dependencies, module structure, wiring, and startup in place. The shape of the system stays the same — Indexer polls and processes, Retriever answers queries, graph.py provides CRUD, vectorstore.py provides search. The internals just become empty stubs waiting to be replaced.
