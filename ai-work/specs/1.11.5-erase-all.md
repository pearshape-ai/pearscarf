# Erase-all script

## Context

- `scripts/reindex_all.py` wipes Neo4j and Qdrant and resets `indexed` flags on Postgres records — but leaves the records themselves intact
- Running a clean eval requires wiping everything: records, graph, and vectors
- A new `scripts/erase_all.py` script is needed that wipes the full system state and then runs reindex
- Follows the same pattern as `reindex_all.py`: confirmation prompt, summary of what will be affected, destructive only on confirmation

## TODO

- New script `scripts/erase_all.py`
- Shows count of what will be deleted before acting: records count, emails count, issues count, issue_changes count, Neo4j node count, Neo4j relationship count, Qdrant vector count
- Requires explicit confirmation before proceeding — same `[y/N]` prompt as `reindex_all.py`
- On confirmation, executes in order:
  - Wipe Neo4j: delete all nodes and relationships
  - Wipe Qdrant: delete and recreate the records collection
  - Wipe Postgres records tables: truncate `issue_changes`, `issues`, `emails`, `records` in dependency order
  - Reset Qdrant and Neo4j client caches (same as `reindex_all.py`)
- After wipe, prints confirmation of what was deleted with counts
- No reindex step — there is nothing left to reindex
- Available as `psc erase-all` CLI command in `cli.py`
- Requires same `[y/N]` confirmation when called via CLI

## What NOT to do

- No automatic execution without confirmation
- No partial wipes — all stores wiped together or not at all
- No changes to `reindex_all.py`
- Do not reset sequences or auto-increment counters — record IDs restart from the next available number naturally
